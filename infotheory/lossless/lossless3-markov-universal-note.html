Lossless 3 — Companion Notes

Stationarity vs Ergodicity

A process is stationary if $\Pr(X_{i_1}\in A_1,\dots,X_{i_k}\in A_k) = \Pr(X_{i_1+t}\in A_1, \dots, X_{i_k+t}\in A_k)$ for all $t$. It is ergodic if every shift-invariant event has probability $0$ or $1$.

For a Markov chain, stationary $+$ irreducible $+$ aperiodic $\Rightarrow$ ergodic. Most slides assume stationary i.i.d. or stationary ergodic Markov; both are routinely satisfied in language modeling.

Entropy Rate — Two Limits Agree (Detail)

Define $H_n = H(X_n\mid X_1^{n-1})$ and $\bar H_n = H(X^n)/n$. By the chain rule, $\bar H_n = (1/n)\sum_{i=1}^n H_i$, so $\bar H_n$ is the Cesàro average of $H_i$.

Monotonicity of $H_n$. Stationarity gives $H(X_n\mid X_1^{n-1}) = H(X_{n-1}\mid X_0, X_1^{n-2}) \le H(X_{n-1}\mid X_1^{n-2}) = H_{n-1}$. So $H_n$ is non-increasing, bounded below by $0$, hence $\lim H_n = H'$.

Cesàro lemma. If $H_n \to H'$, then $\bar H_n \to H'$. Hence $H = H'$.

Two-State Chain — Verifying $\pi$

For the example $\alpha = 0.1$, $\beta = 0.4$:

$$\pi P = (\pi_0, \pi_1)\begin{pmatrix} 0.9 & 0.1 \\ 0.4 & 0.6 \end{pmatrix} = (0.9\pi_0 + 0.4\pi_1,\; 0.1\pi_0 + 0.6\pi_1).$$

Equating to $(\pi_0, \pi_1) = (0.8, 0.2)$: $0.9(0.8) + 0.4(0.2) = 0.72 + 0.08 = 0.80$. $\checkmark$

Universal Coding — Krichevsky–Trofimov

The KT mixture estimator for an i.i.d. source with $|\mathcal{X}| = k$:

$$\hat p_{\mathrm{KT}}(x_t = a \mid x_1^{t-1}) = \frac{n_a(x_1^{t-1}) + 1/2}{t - 1 + k/2},$$

where $n_a$ is the empirical count of symbol $a$ in $x_1^{t-1}$. Adding the Laplace-style $1/2$ pseudocount yields a Bayesian mixture against the Dirichlet$(1/2,\dots,1/2)$ prior.

Plug into arithmetic coding to get a universal code with redundancy $((k-1)/2)\log n + O(1)$ — matching the lower bound up to $O(1)$.

Context Tree Weighting

For Markov sources of unknown order, CTW (Willems–Shtarkov–Tjalkens 1995) maintains a weighted mixture of all KT estimators on all binary contexts up to depth $D$. Achieves redundancy $O(\Gamma\log n / n)$ where $\Gamma$ is the number of leaves in the optimal context tree.

CTW is provably optimal among "tree models" and is competitive with finite-state Markov models when the depth $D$ is large enough.

LZ78 — Why $c(n)\log c(n)/n \to H$

Sketch from Cover & Thomas, Theorem 13.5.3:

  1. Group the LZ78 phrases by length. Phrases of length $\ell$ correspond to depth-$\ell$ leaves in the parsing tree.
  2. Each phrase is the deepest match plus one extension symbol; by the prefix-tree structure, no phrase is repeated.
  3. For a stationary ergodic source: the empirical phrase frequencies converge to the true marginals of $X^\ell$, giving $\sum_\ell c_\ell \log c_\ell / n \to H(\mathscr{X})$.

The standard proof actually uses a tighter argument via the conditional probabilities of suffixes: each LZ78 phrase, on a stationary ergodic source, has self-information $\approx -\log p(\text{phrase}) \approx \ell H$.

LZ77 vs LZ78 — Practical Difference

LZ78 builds an explicit dictionary; output is (index, symbol). Used in compress, GIF (with LZW variant).

LZ77 uses a sliding window: output is (offset, length, symbol), pointing back into the recent history. Used in gzip, deflate, zstd.

LZ77 is much more practical for decompressing on demand — no dictionary state to rebuild. Both are asymptotically optimal.

Modern: LLM-Based Compression

An LLM provides $q(x_t\mid x_1^{t-1})$ for any prefix. Plug into arithmetic coding $\Rightarrow$ rate $H(p) + D(p\,\|\,q)$. As $q\to p$, rate $\to H(p)$.

Recent results (DeepMind 2023, ICML; Delétang et al.) show that large language models compress text better than gzip and even bzip2 — they have learned a more accurate sequential model than classical universal coders. The compression-rate metric is now a standard probe of language model quality.
