01

Sources with Memory

Real text and images are correlated

i.i.d. Is Not Enough

English text: letter frequencies give $H \approx 4.1$ bits/letter, but real English compresses to $\approx 1.3$ bits/letter.

Reason: "q" is almost always followed by "u"; "th" is hugely overrepresented; etc.

Per-letter entropy $H(X) \gg$ conditional $H(X_t \mid X_1^{t-1})$ for $t$ large — correlation reduces uncertainty.

First-Order Markov Process

Definition.

A sequence $X_1, X_2, \dots$ is a first-order Markov chain if for every $i \ge 2$:

$$p_{X_i\mid X_1^{i-1}}(x_i \mid x_1, \dots, x_{i-1}) \;=\; p_{X_i\mid X_{i-1}}(x_i\mid x_{i-1}).$$

The future depends on the past only through the current state.

Joint factors as $p(x^n) = p(x_1)\prod_{i=2}^n p(x_i\mid x_{i-1})$.

Example — Two-State Chain

$\mathcal{X} = \{0, 1\}$. Transition matrix

$$P = \begin{pmatrix} 1-\alpha & \alpha \\ \beta & 1-\beta \end{pmatrix}, \qquad P_{ij} = \Pr(X_{t+1} = j \mid X_t = i).$$

Stationary distribution $\pi = (\pi_0, \pi_1)$ satisfies $\pi P = \pi$:

$$\pi_0 = \frac{\beta}{\alpha+\beta}, \qquad \pi_1 = \frac{\alpha}{\alpha+\beta}.$$

If we start with $X_1\sim\pi$, the chain is stationary.

Joint Entropy of Markov Chain

Apply the chain rule to $p(x^n) = p(x_1)\prod p(x_i\mid x_{i-1})$:

$$H(X^n) \;=\; H(X_1) + \sum_{i=2}^n H(X_i \mid X_{i-1}).$$

For a stationary Markov chain, every conditional $H(X_i\mid X_{i-1})$ equals $H(X_2\mid X_1)$:

$$H(X^n) \;=\; H(X_1) + (n-1)\,H(X_2\mid X_1).$$

$k$-th Order Markov

Definition.

$\{X_i\}$ is $k$-th order Markov if

$$p_{X_i\mid X_1^{i-1}}(x_i\mid x_1^{i-1}) \;=\; p_{X_i\mid X_{i-k}^{i-1}}(x_i\mid x_{i-k}^{i-1}).$$

Joint:

$$p(x^n) \;=\; p(x_1^k) \prod_{i=k+1}^n p(x_i\mid x_{i-k}^{i-1}).$$

Common in language modeling: $n$-gram models are $(n-1)$-th order Markov approximations to text.

02

Entropy Rate

The fundamental limit for sources with memory

Definition — Entropy Rate

Definition.

For a stationary process $\mathscr{X} = \{X_i\}_{i=1}^\infty$, the entropy rate is

$$H(\mathscr{X}) \;=\; \lim_{n\to\infty} \frac{1}{n} H(X_1, \dots, X_n).$$

An equivalent form (when the limit exists):

$$H'(\mathscr{X}) \;=\; \lim_{n\to\infty} H(X_n \mid X_1, \dots, X_{n-1}).$$

Theorem — Limits Exist and Agree

Theorem.

For any stationary process,

$$H(\mathscr{X}) \;=\; H'(\mathscr{X}).$$

Both limits exist (possibly $+\infty$).

Sketch: stationarity gives $H(X_n\mid X_1^{n-1}) \le H(X_n\mid X_2^{n-1}) = H(X_{n-1}\mid X_1^{n-2})$, so the conditional sequence is monotone non-increasing · bounded below by 0 · converges. Cesàro averaging gives the equality.

Stationary Markov — Closed Form

For a stationary first-order Markov chain:

$$\frac{1}{n}H(X^n) \;=\; \frac{H(X_1) + (n-1) H(X_2\mid X_1)}{n} \;\to\; H(X_2\mid X_1).$$

Hence

$$H(\mathscr{X}) \;=\; H(X_2\mid X_1) \;=\; \sum_x \pi(x)\,H(P(\cdot\mid x)),$$

where $\pi$ is the stationary distribution.

Example — Two-State Chain

$\alpha = 0.1$, $\beta = 0.4$. $\pi = (\beta/(\alpha+\beta), \alpha/(\alpha+\beta)) = (0.8, 0.2)$.

  • $H(X_2\mid X_1 = 0) = H_b(0.1) \approx 0.469$
  • $H(X_2\mid X_1 = 1) = H_b(0.4) \approx 0.971$
$$H(\mathscr{X}) \;=\; 0.8 \cdot 0.469 + 0.2\cdot 0.971 \;\approx\; 0.569 \text{ bits/sym}.$$

Compare $H(X_1) = H_b(0.2) \approx 0.722$. Memory reduces entropy by $\sim 0.15$ bits/sym.

Coding a Markov Source

Arithmetic coding works for any sequential model: at each step, partition the current interval according to $p(\cdot\mid x_1^{t-1})$.

For a stationary first-order Markov chain:

$$\mathbb{E}[\ell(X^n)] \;\le\; \log\tfrac{1}{p(X^n)} + 2 \;=\; H(X^n) + 2.$$

Per-symbol rate

$$\frac{\mathbb{E}[\ell(X^n)]}{n} \;\to\; H(\mathscr{X}) \;=\; H(X_2\mid X_1).$$

Markov entropy rate is the operational limit.

03

Universal Coding

When the source distribution is unknown

The Universal Problem

Setup: $X_1, \dots, X_n$ i.i.d. from unknown $P_X\in\mathcal{P}$ where $\mathcal{P}$ is a known class.

A universal code is a single sequence of codes $f_n$ achieving

$$\frac{\mathbb{E}_P[\ell(X^n)]}{n} \;\to\; H(P) \quad\text{for every } P\in\mathcal{P}.$$

The redundancy:

$$R_n(P) \;=\; \frac{\mathbb{E}_P[\ell(X^n)]}{n} - H(P).$$

Two-Pass Scheme

Simple template (works for i.i.d.):

  1. Examine first $m$ symbols, encode with a fixed sub-optimal code.
  2. Both encoder & decoder estimate $\hat P_X$ from $X_1^m$.
  3. Build a Shannon/arithmetic code from $\hat P_X$.
  4. Encode $X_{m+1}, \dots, X_n$ with that code.

Tradeoff: large $m$ $\Rightarrow$ accurate $\hat P$ but more bits wasted up front. Tune $m = O(\sqrt{n})$ or $O(\log n)$.

Redundancy — The Mismatch Cost

Encoding the tail $X_{m+1}^n$ for true $P$ using code designed for $\hat P$:

$$\frac{1}{n - m}\mathbb{E}[\ell(X_{m+1}^n)] \;\approx\; H(P) + D(P\,\|\,\hat P).$$

Empirical $\hat P$ converges to $P$ at rate $\|\hat P - P\| = O(1/\sqrt{m})$, so $D(P\,\|\,\hat P) = O(1/m)$.

Total redundancy: $O(1/m) + (m \cdot c)/n = O(1/\sqrt{n})$ for $m = \sqrt{n}$.

Lower Bound on Redundancy

For the class $\mathcal{P}$ of all i.i.d. distributions on $|\mathcal{X}| = k$ symbols:

$$\inf_{f_n}\;\sup_{P\in\mathcal{P}}\; n R_n(P) \;\ge\; \frac{k-1}{2}\log n - O(1).$$

Optimal universal codes (e.g., Krichevsky–Trofimov mixture, Context Tree Weighting) match this rate.

Universal coding pays only a $\log n$-style overhead — vanishing per-symbol redundancy.

04

Lempel–Ziv

Learn while you encode

The LZ Idea

Two-pass schemes separate learning from encoding. Lempel–Ziv does both at once:

  • Build a dictionary of phrases seen so far.
  • Each new phrase = (existing phrase, extension symbol).
  • Send indices into the dictionary, not raw symbols.

Frequent contexts grow the dictionary · fewer bits per phrase as the file grows.

Ziv and Lempel, "Compression of individual sequences via variable-rate coding", IEEE Trans. Inf. Theory 1978.

LZ78 — The Algorithm

  1. Initialize dictionary $\mathcal{D} = \{\varepsilon\}$ (empty string at index 0).
  2. Find the longest prefix $w$ of the unread input that lies in $\mathcal{D}$.
  3. Let $a$ be the symbol after $w$. Output the pair $(\mathrm{idx}(w), a)$.
  4. Add $wa$ to $\mathcal{D}$ at the next free index.
  5. Advance past $wa$. Repeat until end of input.

Worked Example — "ABBABABAA..."

step$w$$a$outputnew entry
1$\varepsilon$A(0, A)1: A
2$\varepsilon$B(0, B)2: B
3BA(2, A)3: BA
4BAB(3, B)4: BAB
5AA(1, A)5: AA

5 phrases · bits per pair $\approx \lceil\log_2 5\rceil + \lceil\log_2|\mathcal{X}|\rceil$. Improves as the dictionary grows.

Asymptotic Optimality

Theorem (Ziv–Lempel, 1978).

For any stationary ergodic source $\mathscr{X}$ with entropy rate $H(\mathscr{X})$, LZ78 satisfies

$$\frac{\ell_{\mathrm{LZ}}(X_1^n)}{n} \;\xrightarrow{\;n\to\infty\;}\; H(\mathscr{X}) \quad \text{a.s.}$$

No prior knowledge of the source — the algorithm learns the optimal model online.

LZ77 (sliding-window variant) underlies gzip, zlib, PNG, ZIP · LZMA underlies xz.

Proof Idea — LZ78

Let $c(n)$ = number of distinct phrases parsed from $X_1^n$. Each phrase is encoded with $\log c(n) + \log|\mathcal{X}|$ bits, so

$$\ell_{\mathrm{LZ}}(X_1^n) \;\le\; c(n)\bigl(\log c(n) + \log|\mathcal{X}|\bigr).$$

Two key facts:

  1. $c(n) \le n / \log_{|\mathcal{X}|} n$ (Ziv inequality · phrases grow super-logarithmically).
  2. $c(n) \log c(n) / n \to H(\mathscr{X})$ a.s. (using ergodic theory).

Combine to get the rate. $\square$

Practical Note — What Real Codecs Do

  • gzip / zlib / PNG · LZ77 + Huffman on phrase indices
  • xz / lzma · LZ + range (arithmetic) coding
  • bzip2 · Burrows–Wheeler transform + Huffman, not LZ
  • Brotli, Zstandard · LZ + arithmetic + dictionary preset
  • Modern LLM tokenizers · byte-pair encoding inherits the dictionary spirit

All asymptotically achieve $H(\mathscr{X})$ · differences are constants and engineering.

Recap — Lossless Compression Series

  • Single-symbol codes: Kraft, Shannon, Huffman · $L \in [H, H+1)$.
  • AEP & block codes: $R > H$ achievable, $R < H$ impossible.
  • Arithmetic coding: $L \le \log(1/p) + 2$, no codebook.
  • Markov sources: entropy rate $H(\mathscr{X})$ is the limit.
  • Universal coding: achieve $H(P)$ without knowing $P$ · LZ78.
Across the board: entropy is the operational limit, achievable by an algorithm.
