01

AEP

Almost all long sequences look the same

Motivation

$X_1, X_2, \dots$ i.i.d. from $p_X$ on $\mathcal{X}$. There are $|\mathcal{X}|^n$ possible sequences, but most have negligible probability.

Empirical observation. A "long" sequence almost always has empirical frequency $\hat{p}(a)\approx p(a)$ for each $a\in\mathcal{X}$, hence

$$\log p(x^n) \;=\; \sum_a n\hat{p}(a)\log p(a) \;\approx\; -nH(X).$$
"Almost all sequences are equally probable, with probability $\approx 2^{-nH(X)}$."

Theorem — AEP

Theorem (Shannon–McMillan, 1948).

If $X_1, X_2, \dots$ are i.i.d. with finite entropy $H(X)$,

$$-\frac{1}{n}\log p(X_1, \dots, X_n) \;\xrightarrow{\;\text{prob}\;}\; H(X).$$

Proof. $-\tfrac{1}{n}\log p(X^n) = \tfrac{1}{n}\sum_i (-\log p(X_i))$ is an empirical average of i.i.d. terms with mean $H(X)$. Apply the weak law of large numbers. $\square$

Definition — Typical Set

Definition ($\varepsilon$-typical sequence).

$$A_\varepsilon^{(n)} \;=\; \left\{ x^n \in \mathcal{X}^n :\; \left|-\tfrac{1}{n}\log p(x^n) - H(X)\right| \le \varepsilon \right\}.$$

Equivalently, $x^n\in A_\varepsilon^{(n)}$ iff

$$2^{-n(H(X)+\varepsilon)} \;\le\; p(x^n) \;\le\; 2^{-n(H(X)-\varepsilon)}.$$

Three Properties of $A_\varepsilon^{(n)}$

  1. High probability. $\Pr(X^n\in A_\varepsilon^{(n)})\to 1$ as $n\to\infty$.
  2. Bounded size.
    $$|A_\varepsilon^{(n)}| \;\le\; 2^{n(H(X)+\varepsilon)}.$$
  3. Lower bound on size (for $n$ large enough):
    $$|A_\varepsilon^{(n)}| \;\ge\; (1-\varepsilon)\,2^{n(H(X)-\varepsilon)}.$$

Proof — Property 1

By AEP, $-\tfrac{1}{n}\log p(X^n) \to H(X)$ in probability. Equivalently:

$$\Pr\!\left\{\left|-\tfrac{1}{n}\log p(X^n) - H(X)\right| > \varepsilon\right\} \;\to\; 0.$$

The complement of this event is exactly $\{X^n\in A_\varepsilon^{(n)}\}$, so $\Pr(X^n\in A_\varepsilon^{(n)}) \to 1$. $\square$

Proof — Property 2 (Upper Bound)

Each typical sequence has $p(x^n) \ge 2^{-n(H+\varepsilon)}$, so

$$1 \;\ge\; \Pr(X^n\in A_\varepsilon^{(n)}) \;=\; \sum_{x^n\in A_\varepsilon^{(n)}} p(x^n) \;\ge\; |A_\varepsilon^{(n)}|\,2^{-n(H+\varepsilon)}.$$

Solving:

$$|A_\varepsilon^{(n)}| \;\le\; 2^{n(H+\varepsilon)}. \qquad\square$$

Proof — Property 3 (Lower Bound)

Each typical sequence has $p(x^n) \le 2^{-n(H-\varepsilon)}$, so for $n$ large enough that $\Pr(A_\varepsilon^{(n)}) \ge 1-\varepsilon$:

$$1-\varepsilon \;\le\; \Pr(X^n\in A_\varepsilon^{(n)}) \;\le\; |A_\varepsilon^{(n)}|\,2^{-n(H-\varepsilon)}.$$

Hence

$$|A_\varepsilon^{(n)}| \;\ge\; (1-\varepsilon)\,2^{n(H-\varepsilon)}. \qquad\square$$

Picture — Volume Counting

The full space $\mathcal{X}^n$ has $|\mathcal{X}|^n = 2^{n\log|\mathcal{X}|}$ sequences.

The typical set has $\approx 2^{nH(X)}$ sequences and probability $\to 1$.

Atypical sequences vastly outnumber typical ones (when $p$ is non-uniform), but their total probability vanishes.

Example — Bernoulli Source

$X_i\sim\mathrm{Bern}(0.1)$. $H(X) = H_b(0.1) \approx 0.469$ bits. $n = 100$.

  • Total sequences: $2^{100} \approx 1.27\times 10^{30}$.
  • Typical set size: $\approx 2^{nH} = 2^{46.9} \approx 1.3\times 10^{14}$.
  • Typical sequences have between $\approx 7$ and $\approx 13$ ones (heuristic; exact range depends on $\varepsilon$).

Compression ratio $\approx 100/47 \approx 2.13\times$.

02

Source Coding Theorem

Bits per symbol $\to H(X)$, no lower

Block Code — Setup

  • Encoder $f_n: \mathcal{X}^n \to \{0,1\}^*$ · rate $R = \log|\mathrm{range}(f_n)| / n$.
  • Decoder $g_n: \{0,1\}^* \to \mathcal{X}^n$.
  • Error: $P_e^{(n)} = \Pr(g_n(f_n(X^n)) \ne X^n)$.

$R$ is achievable if there is a sequence of codes with

$$\frac{|f_n|_{\max}}{n} \le R, \qquad P_e^{(n)} \to 0.$$

Theorem — Shannon's Source Coding

Theorem (Shannon, 1948).

For an i.i.d. source with entropy $H(X)$:

  • Achievability: any rate $R > H(X)$ is achievable.
  • Converse: any rate $R < H(X)$ is not achievable.

Hence the operational compression limit equals the entropy.

Shannon, "A Mathematical Theory of Communication", BSTJ 1948.

Achievability — Idea

Pick $\varepsilon$ with $H(X) + \varepsilon < R$. Build the code:

  1. Enumerate the typical set $A_\varepsilon^{(n)}$. Need at most $2^{n(H+\varepsilon)} \le 2^{nR}$ indices.
  2. Encoder. If $x^n\in A_\varepsilon^{(n)}$, output its index padded to $\lceil nR\rceil$ bits prepended with a $0$. Else output $\texttt{1}\,\underbrace{00\cdots 0}_{\lceil nR\rceil}$.
  3. Decoder. Strip leading bit; if $0$, look up index in the typical set; else declare error.

Achievability — Error Goes to Zero

Error happens only on the atypical set:

$$P_e^{(n)} \;=\; \Pr(X^n\notin A_\varepsilon^{(n)}) \;\to\; 0$$

by AEP (Property 1). Rate is

$$\frac{1 + \lceil nR\rceil}{n} \;\to\; R.$$

Hence any $R > H(X)$ is achievable. $\square$

Converse — $R < H(X)$ Fails

Fix $R < H(X) - \delta$. Code maps $X^n$ to one of $\le 2^{nR}$ outputs · only $\le 2^{nR}$ sequences are decodable correctly. Call this set $B$.

For any $\eta > 0$, the typical set $A_\eta^{(n)}$ has $|A_\eta^{(n)}|\ge (1-\eta) 2^{n(H - \eta)}$. Pick $\eta < \delta/2$:

$$|B| \;\le\; 2^{nR} \;\le\; 2^{n(H-\delta)} \;\ll\; |A_\eta^{(n)}|.$$

Most typical sequences are not in $B$. Continue on next slide.

Converse — Continued

$$\begin{aligned} P_e^{(n)} &\;\ge\; \Pr(X^n\in A_\eta^{(n)}\setminus B) \\ &\;=\; \sum_{x^n\in A_\eta^{(n)}\setminus B} p(x^n) \\ &\;\ge\; (|A_\eta^{(n)}| - |B|)\,2^{-n(H+\eta)} \\ &\;\ge\; \bigl((1-\eta) 2^{n(H-\eta)} - 2^{nR}\bigr)\,2^{-n(H+\eta)} \\ &\;\to\; 1. \end{aligned}$$

$P_e^{(n)} \not\to 0$. $\square$

What If We Want Zero Error?

Allow only uniquely decodable codes (no error). Block lengths must satisfy

$$\mathbb{E}[\ell(X^n)] \;\ge\; H(X^n) \;=\; n H(X).$$

Achievable: Shannon code on $X^n$ gives $L(X^n) < H(X^n) + 1 = nH(X) + 1$.

Per-symbol rate $\le H(X) + 1/n \to H(X)$. Zero-error compression also approaches $H$.

03

Arithmetic Coding

Encode the entire sequence as one number

Why Arithmetic Coding?

Block Shannon/Huffman on $X^n$ achieves $L \le H(X^n) + 1$. But the codebook has $|\mathcal{X}|^n$ entries — exponential storage.

Arithmetic coding achieves the same bound with $O(n)$ time and $O(1)$ working memory:

  • Map $x^n$ to a sub-interval of $[0,1)$ of length $p(x^n)$.
  • Send a binary expansion identifying that interval.
  • Length of that expansion $\le \log\tfrac{1}{p(x^n)} + 2$ bits.

Construction — Recursive Subdivision

Order the alphabet $\mathcal{X} = \{a_1 < a_2 < \dots\}$. Cumulative distribution $F(a_k) = \sum_{j\le k} p(a_j)$.

Start with $[L_0, H_0) = [0, 1)$. Read symbol $x_t$:

$$\begin{aligned} L_t &= L_{t-1} + (H_{t-1} - L_{t-1}) F(x_t^-) \\ H_t &= L_{t-1} + (H_{t-1} - L_{t-1}) F(x_t). \end{aligned}$$

$F(x_t^-)$ is cumulative just before $x_t$. After $n$ symbols: interval $[L_n, H_n)$ has length $H_n - L_n = p(x^n)$.

Example — Encoding "AB"

$\mathcal{X} = \{A, B, C\}$ with $p = (0.25, 0.15, 0.6)$. Cumulative: $F(A) = 0.25$, $F(B) = 0.40$.

Step 0: $[0, 1)$.

Read $A$: subdivide $[0, 1) \to [0, 0.25)$.

Read $B$: within $[0, 0.25)$, subdivide proportionally to $p$:

$$[\,0,\;0.0625)\;[\,0.0625,\;0.10)\;[\,0.10,\;0.25\,).$$

$B$ corresponds to the middle: $[0.0625, 0.10)$. Length $= 0.0375 = 0.25 \cdot 0.15 = p(AB)$. $\;\checkmark$

From Interval to Bits

Pick any $z\in[L_n, H_n)$, e.g. $z = (L_n + H_n)/2$. Truncate its binary expansion to $K$ bits, $\hat z = 0.b_1 b_2\cdots b_K$, choosing $K$ so that

$$L_n \;\le\; \hat z \;<\; \hat z + 2^{-K} \;<\; H_n.$$

The truncated $\hat z$ uniquely identifies the interval $[L_n, H_n)$.

Decoder: given $\hat z$, retrace the subdivisions to recover $x^n$.

Example — Decoding

Continuing: $[L_n, H_n) = [0.0625, 0.10)$. Midpoint $z = 0.08125$.

Binary: $0.08125 \approx 0.00010100\ldots_{(2)}$. Try $K = 4$: $\hat z = 0.0001_{(2)} = 0.0625$. Then $\hat z + 2^{-4} = 0.0625 + 0.0625 = 0.125 \not< 0.10$.

$K = 5$: $\hat z = 0.00010_{(2)} = 0.0625$. $\hat z + 2^{-5} = 0.09375 < 0.10$. $\checkmark$ Send $\texttt{00010}$ (5 bits).

$\log\tfrac{1}{p(AB)} = \log\tfrac{1}{0.0375} \approx 4.74$, so we paid $\le \log\tfrac{1}{p} + 2 = 6.74$ bits. We used 5. $\checkmark$

04

Arithmetic Coding — Optimality

$L(X^n) \le H(X^n) + 2$

Lemma — Truncation Length

Lemma.

Let $\hat z$ be the binary truncation chosen by the procedure. Then

$$2^{-K} \;\ge\; \tfrac{1}{4}(H_n - L_n) \;=\; \tfrac{1}{4} p(x^n).$$

Hence $K \le \log\tfrac{4}{p(x^n)} = \log\tfrac{1}{p(x^n)} + 2$.

Proof — Truncation Length

$K$ is the smallest integer satisfying $L_n\le \hat z < \hat z + 2^{-K} < H_n$. So with $K-1$ bits, the condition fails: either $\tilde z < L_n$ or $\tilde z + 2^{-(K-1)} > H_n$.

Case 1: $\tilde z < L_n$. Since the smaller truncation $\tilde z$ is at distance $\le 2^{-(K-1)}$ from valid $\hat z$, and $\hat z\ge L_n$, we have $\tilde z + 2^{-(K-1)} > L_n$. Combined with $\tilde z + 2^{-(K-1)} > (L_n + H_n)/2$ (from failing the right side), we get

$$2^{-(K-1)} \;>\; \tfrac{1}{2}(H_n - L_n) \;\Rightarrow\; 2^{-K} \;>\; \tfrac{1}{4}(H_n - L_n).$$

Case 2 ($\tilde z + 2^{-(K-1)} > H_n$) is symmetric. $\square$

Theorem — Arithmetic Coding Bound

Theorem.

For an i.i.d. (or any product-distributed) source, arithmetic coding produces a codeword of length

$$\ell(x^n) \;\le\; \log\tfrac{1}{p(x^n)} + 2.$$

Hence

$$\mathbb{E}[\ell(X^n)] \;\le\; H(X^n) + 2,\qquad \frac{\mathbb{E}[\ell(X^n)]}{n} \;\le\; H(X) + \frac{2}{n}.$$

Comparison — Three Codes

Arithmetic coding pays one extra bit per block to avoid the codebook entirely.

In practice arithmetic coding is the workhorse: gzip, JPEG, modern LLM tokenizers' compression layers.

Recap

  • AEP: $-\tfrac{1}{n}\log p(X^n)\to H(X)$ in probability.
  • Typical set: $\approx 2^{nH}$ sequences with total probability $\to 1$.
  • Source coding theorem: $R > H$ achievable, $R < H$ impossible.
  • Arithmetic coding: length $\le \log(1/p) + 2$, no codebook.
  • Per-symbol rate $\to H(X)$ for any practical scheme.
