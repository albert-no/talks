Lossless 1 — Companion Notes

Uniquely Decodable vs Prefix-Free

Every prefix-free code is uniquely decodable, but not every UD code is prefix-free. Example: the code $f(A) = 0$, $f(B) = 01$, $f(C) = 011$ is uniquely decodable (decode by reverse-parsing — "$0$" can only end a codeword) but not prefix-free.

By the Kraft–McMillan theorem, however, any UD code can be replaced by a prefix-free code with the same length distribution. So restricting to prefix-free codes loses no rate, only flexibility.

Kraft Sufficiency — Greedy Construction

Sort lengths $\ell_1\le\ell_2\le\dots\le\ell_n$. Walk a depth-$\ell_n$ binary tree from left to right; at each step, claim the next available leaf at depth $\ell_i$, and prune the subtree above it.

The leaf-budget at step $i$ is $2^{\ell_n} - \sum_{j<i} 2^{\ell_n - \ell_j}$. By Kraft, $\sum_j 2^{-\ell_j}\le 1$, so this is $\ge 2^{\ell_n - \ell_i}$, which is exactly the number of depth-$\ell_n$ leaves under any node at depth $\ell_i$. Hence the greedy step succeeds.

Kraft–McMillan — Why $T^k$?

The trick is to consider $k$-fold extensions: encode $k$ source symbols at once. The encoded bit string has length between $k$ and $k\ell_{\max}$. Unique decodability of the original code implies unique decodability of the $k$-fold extension, so distinct $k$-tuples have distinct bit strings.

Counting: $A_m \le 2^m$ (binary strings of length $m$). Hence $T^k = \sum_m A_m 2^{-m} \le k\ell_{\max}$. Since this holds for every $k$, $T \le 1$ (or else $T^k$ would grow exponentially).

Entropy Bound on UD Codes

The cleaner derivation: define $q_i = 2^{-\ell_i}/T$ where $T = \sum_j 2^{-\ell_j} \le 1$. Then $q$ is a pmf and

$$L - H = \sum_i p_i \ell_i + \sum_i p_i \log p_i = \sum_i p_i \log\frac{p_i}{2^{-\ell_i}} = D(p\,\|\,q) - \log T \ge 0,$$

since $D \ge 0$ and $\log T \le 0$.

Shannon Code — Worst Case

Shannon code length $\lceil \log 1/p \rceil$ can exceed $\log 1/p$ by up to nearly $1$ bit per symbol for skewed distributions. Worst case: $p = (1-\epsilon, \epsilon)$ for small $\epsilon$. Shannon assigns $\ell = (1, \lceil\log 1/\epsilon\rceil)$, giving $L \approx 1$ but $H \approx \epsilon\log(1/\epsilon)\to 0$. Gap $\approx 1$ bit.

Huffman fixes this: it would assign $\ell = (1,1)$, giving $L = 1$, but with the same gap $\approx 1$ from $H$. Both codes hit the $1$-bit gap; only block coding bridges it.

Mismatch Cost — Operational Reading

$L_{\hat p}(p) = H(p) + D(p\,\|\,\hat p)$ is the rate when we encode with code designed for $\hat p$ but the truth is $p$. Two important readings:

  • Universal coding: if $\hat p$ is an estimate of $p$, the redundancy is $D(p\,\|\,\hat p)$, going to 0 as $\hat p \to p$.
  • Cross-entropy as training loss: in language modeling, the negative log-likelihood under the model $\hat p$ averaged over the true data $p$ is exactly $H(p) + D(p\,\|\,\hat p)$. Minimizing this over $\hat p$ minimizes $D(p\,\|\,\hat p)$ — the model's compression rate gap from optimal.

Huffman — Optimality Proof Detail

Lemma 1 (probability/length monotone) is proven by exchange: swap codewords of any pair $(x,x')$ with $p(x)>p(x')$ and $\ell(x)>\ell(x')$, decreasing $L$ by $(p(x)-p(x'))(\ell(x)-\ell(x')) > 0$.

Lemma 2 (least-probable two are siblings): if the two least-probable symbols $a_{n-1}, a_n$ sit at maximum depth $\ell_{\max}$ but are not siblings, $a_n$'s sibling slot is unused. Detach $a_n$, attach it as $a_{n-1}$'s sibling. The new code remains prefix-free and has $a_n$'s length unchanged or decreased — never worse.

Inductive step: merging $a_{n-1}, a_n$ into a single super-symbol $a^*$ with $p(a^*) = p(a_{n-1}) + p(a_n)$ reduces alphabet to size $n-1$. The resulting smaller-alphabet problem has a unique optimal solution by induction; lift it back by splitting $a^*$ into siblings.

Block Coding — How Block Length Trades Off

Encoding $X^n$ as a single super-symbol with Shannon code: $L(X^n) \le H(X^n) + 1 = nH(X) + 1$ (i.i.d.). Per-symbol rate $\le H(X) + 1/n$.

Block coding cost: codebook size $|\mathcal{X}|^n$ (exponential), encoding/decoding lookup table grows exponentially. Hence Huffman blocks are impractical beyond $n \approx 6\ldots 8$ for $|\mathcal{X}|\sim 100$. Arithmetic coding (Lecture 2) achieves the same bound without storing a codebook.
