01

Codes and Rate

What does "compression" mean?

Lossless Source Coding — Setup

Source $X_1, X_2, \dots$ on alphabet $\mathcal{X}$. Encoder maps each $x\in\mathcal{X}$ to a binary codeword $f(x) \in \{0,1\}^*$.

Stream of symbols $\to$ concatenation of codewords:

$$x_1 x_2 x_3 \cdots \;\longmapsto\; f(x_1)\,f(x_2)\,f(x_3)\cdots$$

Goal. Use as few bits per symbol as possible while keeping decoding unambiguous.

Expected Rate

Codeword length $\ell(x) = |f(x)|$. The expected rate:

$$L(f) \;=\; \mathbb{E}[\ell(X)] \;=\; \sum_{x\in\mathcal{X}} p_X(x)\,\ell(x) \quad\text{(bits/symbol)}.$$

Three constraints we will impose:

  • Unambiguous decoding · no two source streams encode to the same bit string.
  • Prefix-free · no codeword is a prefix of another.
  • Optimality · minimize $L$ over all permitted codes.

Example — Uniquely Decodable

$\mathcal{X} = \{A, B, C\}$ with

$$f(A) = \texttt{0},\quad f(B) = \texttt{10},\quad f(C) = \texttt{11}.$$

Encode "ABAC": 0·10·0·11 = 010011.

Decode greedily, reading left to right: each codeword starts where the previous one ends. Prefix-free, hence uniquely decodable.

Example — Not Uniquely Decodable

Same alphabet, different code:

$$f(A) = \texttt{0},\quad f(B) = \texttt{1},\quad f(C) = \texttt{10}.$$

"ABAC" $\to$ 01010, but so does "BABAB" if we re-parse. The decoder cannot tell.

$f(B) = \texttt{1}$ is a prefix of $f(C) = \texttt{10}$ — this code is not prefix-free.

Prefix-Free Codes as Trees

A binary prefix-free code corresponds to placing each codeword on a leaf of a binary tree:

  • Path from root: read bits left-to-right.
  • No leaf is on the path of another · prefix-free.
  • Length of codeword = depth of leaf.
02

Kraft Inequality

Which codeword lengths are realizable?

Theorem — Kraft's Inequality

Theorem (Kraft, 1949).

A set of codeword lengths $\ell_1, \dots, \ell_n$ corresponds to some prefix-free binary code if and only if

$$\sum_{i=1}^n 2^{-\ell_i} \;\le\; 1.$$

Existence statement · tells us which length-tuples are achievable, without exhibiting the code.

Kraft, MS thesis, MIT 1949.

Proof — ($\Rightarrow$) Necessity

Sort lengths $\ell_1 \le \dots \le \ell_n$. Embed in the full binary tree of depth $\ell_n$.

Each codeword $i$ occupies a node at depth $\ell_i$, blocking off all its $2^{\ell_n - \ell_i}$ depth-$\ell_n$ descendants.

Prefix-free $\Rightarrow$ blocked subtrees are disjoint:

$$\sum_{i=1}^n 2^{\ell_n - \ell_i} \;\le\; 2^{\ell_n}.$$

Divide by $2^{\ell_n}$. $\square$

Proof — ($\Leftarrow$) Sufficiency

Sort $\ell_1 \le\dots\le\ell_n$. Greedy assignment:

  1. Assign $f(1) =$ the leftmost depth-$\ell_1$ leaf.
  2. Prune the entire subtree rooted at $f(1)$.
  3. Assign $f(2) =$ the leftmost remaining depth-$\ell_2$ leaf, prune.
  4. Continue.

Available leaf budget at each step: $2^{\ell_n} - \sum_{j<i} 2^{\ell_n - \ell_j}$. The Kraft inequality guarantees this is $\ge 2^{\ell_n - \ell_i}$, so the greedy step succeeds. $\square$

Theorem — Kraft–McMillan

Theorem (McMillan, 1956).

Any uniquely decodable code (not necessarily prefix-free) satisfies

$$\sum_i 2^{-\ell_i} \;\le\; 1.$$

Implication. Allowing non-prefix-free uniquely decodable codes does not let us reach shorter lengths — we may as well restrict to prefix-free codes.

McMillan, "Two inequalities implied by unique decipherability", IEEE Trans. Inf. Theory 1956.

Proof — Kraft–McMillan

Set $T = \sum_i 2^{-\ell_i}$. For any $k\ge 1$:

$$T^k \;=\; \sum_{i_1, \dots, i_k} 2^{-(\ell_{i_1} + \dots + \ell_{i_k})} \;=\; \sum_{m=k}^{k\ell_{\max}} A_m\, 2^{-m},$$

where $A_m$ counts $k$-tuples of source symbols whose concatenated codewords have total length $m$.

Unique decodability $\Rightarrow$ each binary string of length $m$ is used at most once $\Rightarrow$ $A_m \le 2^m$.

$$T^k \;\le\; \sum_{m=k}^{k\ell_{\max}} 1 \;\le\; k\ell_{\max}.$$

Hence $T \le (k\ell_{\max})^{1/k} \to 1$ as $k\to\infty$, so $T \le 1$. $\square$

Corollary — Lower Bound on Rate

Corollary.

Every uniquely decodable code satisfies

$$L(f) \;=\; \mathbb{E}[\ell(X)] \;\ge\; H(X).$$

Proof. Set $q_i = 2^{-\ell_i} / T$ where $T = \sum 2^{-\ell_j}\le 1$. Then $q$ is a pmf and

$$L - H(X) \;=\; \sum_i p_i(\ell_i - \log\tfrac{1}{p_i}) \;=\; D(p\,\|\,q) - \log T \;\ge\; 0. \;\square$$
03

Shannon Code

$\lceil \log 1/p \rceil$ and the cost of mismatch

Shannon Code — Definition

For each $x\in\mathcal{X}$, set the codeword length

$$\ell(x) \;=\; \left\lceil \log\frac{1}{p_X(x)} \right\rceil.$$

This satisfies Kraft:

$$\sum_x 2^{-\ell(x)} \;\le\; \sum_x 2^{-\log(1/p_X(x))} \;=\; \sum_x p_X(x) \;=\; 1.$$

By Kraft, a prefix-free code with these lengths exists.

Theorem — Entropy Bound

Theorem.

The Shannon code achieves rate

$$H(X) \;\le\; L(\text{Shannon}) \;<\; H(X) + 1.$$

Proof. Use $\log\tfrac{1}{p} \le \lceil\log\tfrac{1}{p}\rceil < \log\tfrac{1}{p} + 1$ termwise:

$$\sum_x p(x)\log\tfrac{1}{p(x)} \;\le\; \sum_x p(x)\,\ell(x) \;<\; \sum_x p(x)\log\tfrac{1}{p(x)} + 1. \;\square$$

Example — Dyadic Distribution

$\mathcal{X} = \{a, b, c, d\}$ with $p = (1/2, 1/4, 1/8, 1/8)$.

Shannon lengths $\ell = (1, 2, 3, 3)$.

$x$$p(x)$$\ell(x)$code
$a$$1/2$$1$0
$b$$1/4$$2$10
$c$$1/8$$3$110
$d$$1/8$$3$111

$L = \tfrac{1}{2}+\tfrac{2}{4}+\tfrac{3}{8}+\tfrac{3}{8} = 1.75 = H(X)$. Exact.

Example — Non-Dyadic

$\mathcal{X} = \{a, b, c\}$ with $p = (1/3, 1/3, 1/3)$.

Shannon lengths $\lceil\log 3\rceil = 2$ for all symbols. Hence $L = 2$.

Compare $H(X) = \log_2 3 \approx 1.585$. Gap $\approx 0.415$ bits / symbol.

Why so wasteful? With non-dyadic probabilities, $\lceil\log 1/p\rceil$ rounds up by up to $1 - \log 1/p\bmod 1$ bits.

Fix: encode blocks of $n$ symbols. Per-symbol overhead $\to 1/n$. (Lecture 2.)

Mismatch — Coding for the Wrong Distribution

Suppose the code is designed for $q$ (lengths $\ell(x) = \log\tfrac{1}{q(x)}$, ignoring ceiling for clarity), but the true source is $p$.

$$\begin{aligned} \mathbb{E}_p[\ell(X)] &= \sum_x p(x) \log\tfrac{1}{q(x)} \\ &= \sum_x p(x) \log\tfrac{p(x)}{q(x)} + \sum_x p(x) \log\tfrac{1}{p(x)} \\ &= D(p\,\|\,q) + H(p). \end{aligned}$$
Mismatch penalty $= D(p\,\|\,q)$ extra bits per symbol.

Example — Mismatch Cost

True source $p = (0.5, 0.25, 0.125, 0.125)$, but model $q = (0.4, 0.4, 0.1, 0.1)$.

Compute $D(p\,\|\,q)$:

$$\begin{aligned} D(p\,\|\,q) &= 0.5\log\tfrac{0.5}{0.4} + 0.25\log\tfrac{0.25}{0.4} + 0.125\log\tfrac{0.125}{0.1}\cdot 2 \\ &\approx 0.161 - 0.170 + 0.080 \approx 0.071 \text{ bits/sym}. \end{aligned}$$

Optimal code $L = H(p) = 1.75$. Mismatched code $L \approx 1.821$.

Misestimating the source by 7% in TV costs ~7% in compression.

04

Huffman Code

Optimal symbol-by-symbol prefix-free code

The Huffman Algorithm

  1. Take the two least probable symbols.
  2. Merge them into a composite with probability $= $ sum.
  3. Repeat on the reduced alphabet until one composite remains.
  4. Read off codewords from the resulting tree (root $\to$ leaf).

Bottom-up tree construction · produces an optimal prefix-free code (Theorem below).

Huffman, "A Method for the Construction of Minimum-Redundancy Codes", Proc. IRE 1952.

$H(X) = -\sum p\log p \approx 2.122$ bits. Gap $\approx 0.078$ bits.

Optimality — Two Lemmas

Lemma 1.

Any optimal prefix-free code can be assumed to satisfy

$$p(x) > p(x') \;\Longrightarrow\; \ell(x) \le \ell(x').$$

Lemma 2.

An optimal prefix-free code can be chosen so that the two least-probable symbols have codewords of equal length, differing in the last bit only.

Proof — Lemma 1 (Sketch)

Suppose $p(x) > p(x')$ but $\ell(x) > \ell(x')$ in some optimal code. Swap their codewords.

Change in expected length:

$$\Delta L = (p(x) - p(x'))(\ell(x') - \ell(x)) \;<\; 0,$$

strictly decreasing $L$ — contradicts optimality. $\square$

Proof — Lemma 2 (Sketch)

Let $a_{n-1}, a_n$ be the two least-probable symbols. By Lemma 1, both have maximum codeword length $\ell_{\max}$.

If their codewords are not siblings in the tree, $a_n$'s sibling slot is empty, so we can shorten $a_n$'s codeword by $1$ — contradicting optimality.

Hence we can take $a_{n-1}, a_n$ as siblings: codewords of equal length differing in the last bit. $\square$

Theorem — Huffman is Optimal

Theorem.

Among all prefix-free codes for $p$, Huffman minimizes the expected length.

Proof (induction on $|\mathcal{X}|$). Base case $|\mathcal{X}|=2$: lengths $(1,1)$, optimal.

Step. Let $a_{n-1}, a_n$ be two least-probable symbols. Lemma 2: they are siblings in some optimal code. Merging them into $a^* = a_{n-1}\cup a_n$ reduces alphabet size by $1$. Optimal-by-induction code on the merged alphabet, then split $a^*$ back into siblings, is optimal on the original. $\square$

Huffman vs Entropy — The 1-Bit Gap

Huffman achieves $L \le H + 1$ (by sandwiching with Shannon). Tight for skewed distributions:

$\mathcal{X} = \{a, b\}$ with $p_a = 0.99$. Shannon $\ell = (1, 7)$. Huffman $\ell = (1, 1)$. Both give $L = 1$, but $H \approx 0.081$ — gap nearly $0.92$ bit.

Symbol-by-symbol coding cannot beat $H+1$. To approach $H$ we must code in blocks — Lecture 2.

Block Coding — Preview

Encode $X^n = (X_1,\dots,X_n)$ as a single super-symbol with pmf $p^{\otimes n}$.

Shannon code on super-symbols satisfies

$$H(X^n) \;\le\; \mathbb{E}[\ell(X^n)] \;<\; H(X^n) + 1.$$

Per-symbol rate:

$$\frac{H(X^n)}{n} \;\le\; \frac{\mathbb{E}[\ell(X^n)]}{n} \;<\; \frac{H(X^n)}{n} + \frac{1}{n}.$$

For i.i.d. source: $H(X^n) = nH(X)$, so per-symbol rate $\to H(X)$. Cost: codebook size $|\mathcal{X}|^n$.

Recap

  • Kraft: length tuple realizable as prefix-free $\iff$ $\sum 2^{-\ell_i}\le 1$.
  • Kraft–McMillan: uniquely decodable codes obey the same bound.
  • Entropy bound: $L\ge H(X)$ for any UD code.
  • Shannon code: $\ell = \lceil\log 1/p\rceil$ · $H \le L < H + 1$.
  • Mismatch: coding for $q$ when truth is $p$ costs $D(p\,\|\,q)$ bits.
  • Huffman: optimal symbol-by-symbol · still up to $1$-bit gap from $H$.
