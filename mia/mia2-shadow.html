Shokri et al. 2017

MIA against black-box ML-as-a-service

Shokri et al. 2017

Shokri et al.: Paper Overview

First systematic black-box MIA.

  • Targets: Google Prediction API, Amazon ML
  • Black-box: confidence vectors only
  • Introduced shadow model methodology
  • Up to 93% attack accuracy
Shokri, Stronati, Song, Shmatikov, "Membership Inference Attacks Against ML Models", IEEE S&P 2017
Shokri et al. 2017

Key Insight

Shadow models mimic the target. Controlled data yields labeled (prediction, in/out) pairs for an attack classifier.

Unsupervised problem reduced to supervised classification.

Shokri et al. 2017

Attack Architecture

Target $f_\theta$ (black-box)
→
Query with $x$
→
Confidence $\sigma(f_\theta(x))$
Shadow Models $f_{\theta_1}, \ldots, f_{\theta_k}$
→
Labeled Data
→
Attack Model $\mathcal{A}_c$

Three stages: train shadows → collect attack data → train attack classifier.

Shokri et al. 2017

Detailed Pipeline

  • Shadow data: datasets $D_1^s, \ldots, D_k^s$ from a similar distribution
  • Train shadows: $k$ models $f_{\theta_1}, \ldots, f_{\theta_k}$
  • Collect predictions: shadows queried on train (in) and held-out (out)
  • Attack dataset: $\{(\sigma(f_{\theta_j}(x)), \text{in/out})\}$
  • Attack model: binary classifier $\mathcal{A}_c$ per class $c$
  • Inference: query target, feed $\sigma(f_\theta(x))$ to $\mathcal{A}_{f_\theta(x)}$
Shokri et al. 2017

Shadow Model Training

$k$ shadow models approximating target behavior:

Requirements

  • Similar data distribution
  • Same or similar architecture
  • Same training procedure (epochs, optimizer)

Why It Works

  • Similar overfitting patterns
  • Member vs. non-member confidence gap: generalizable signal
  • Cross-model membership signature
Shokri et al. 2017

Three Shadow Data Strategies

Model-Based

Synthesize via hill-climbing on target.

Statistics-Based

Sample from public marginals.

Noisy Real Data

Related-domain or noisy public.

Later (Salem 2019): even unrelated data works.

Shokri et al. 2017

Attack Data Collection

For each shadow model $f_{\theta_j}$:

Member Queries

Query on $x \in D_j^s$

Record $(\sigma(f_{\theta_j}(x)), \texttt{IN})$

Non-Member Queries

Query on $x \notin D_j^s$

Record $(\sigma(f_{\theta_j}(x)), \texttt{OUT})$

Labeled attack dataset from shadow member/non-member gap.

Assumption: target gap mirrors shadow gap.

Shokri et al. 2017

Per-Class Attack Models

One binary classifier per class $c$:

$$\mathcal{A}_c : \mathbb{R}^{|\mathcal{C}|} \to \{0, 1\}$$

Input: confidence $\sigma(f_\theta(x))$ for class-$c$ predictions

Output: member (1) or non-member (0)

Per-class: confidence distributions differ by class. Class-specific calibration required.

Shokri et al. 2017

Attack Inference

Given target point $(x, y)$ and model $f_\theta$:

  • Query: $\mathbf{p} = \sigma(f_\theta(x))$
  • Predicted class: $c = \arg\max_i \mathbf{p}_i$
  • Attack: $\hat{m} = \mathcal{A}_c(\mathbf{p})$
  • If $\hat{m} = 1$: declare member

Classifier learns per-class confidence pattern separating members from non-members.

Shokri et al. 2017

Results: Amazon ML

Attack generalizes to Amazon ML.

  • Effective across different ML platforms
  • Model-type-agnostic
  • More classes: higher vulnerability
  • More overfitting: higher vulnerability

Platform-agnostic. Overfit confidence is a universal signature.

Shokri et al. 2017

Class Count and Vulnerability

Output lives on the probability simplex $\Delta^{C-1}$:

$$\sigma(f_\theta(x)) \in \Delta^{C-1} = \left\{\mathbf{p} \in \mathbb{R}^C : p_i \geq 0, \sum_i p_i = 1\right\}$$

Few classes (C=2)

Scalar output.

Limited signal.

Many classes (C=100)

99-dim vector. Much richer membership signature.

More dimensions → richer fingerprint → easier discrimination.

Shokri et al. 2017

Limitations of Shadow Models

  • Data distribution: needs similar-distribution data
  • Architecture knowledge: best when shadow matches target
  • Cost: $k$ models, $k\times$ training
  • Per-class classifiers: poor scaling
  • No theory: empirical only, no optimality guarantees

Open question: drop the assumptions? (Yes: Salem, Sablayrolles.)

Shokri et al. 2017

Pseudocode: Shadow Model Attack

// Phase 1: Build attack training data for j = 1 to k: D_s[j] ← sample_data(distribution) f[j] ← train_model(D_s[j]) for x in D_s[j]: // members attack_data.add(f[j](x), "IN", class(x)) for x in holdout[j]: // non-members attack_data.add(f[j](x), "OUT", class(x)) // Phase 2: Train attack classifiers for c in classes: A[c] ← train_binary_classifier(attack_data[c]) // Phase 3: Attack def attack(target_model, x): p ← target_model(x) c ← argmax(p) return A[c](p) // 1 = member, 0 = non-member

Confidence Threshold Baseline

Simplest attack, pre-shadow:

$$\mathcal{A}(f_\theta, x) = \mathbf{1}\left[\max_c f_\theta(x)_c > \tau\right]$$

High confidence on $x$ → probably a member.

Advantages

  • No shadows
  • No training
  • Single threshold

Limitations

  • Ignores class structure
  • Uses only max, not full vector
  • Weak on calibrated models

Often competitive with full shadow pipeline (Salem et al. 2019).

LOGAN: Generative Models

MIA for GANs (Hayes et al. 2019)

Hayes et al. 2017

LOGAN: MIA Against Generative Models

Shokri: classifiers.

Hayes et al.: MIA on GANs.

Challenge: no confidence vector. Membership definition requires new signals.

  • Memorization: GAN outputs close to training examples
  • Discriminator score as membership signal
  • Reconstruction error as membership signal
Hayes et al. 2017

LOGAN: Attack Architecture

Discriminator-Based

Discriminator $D(x)$ as signal.

$$\mathcal{A}(x) = \mathbf{1}[D(x) > \tau]$$

Real-vs-fake score encodes membership.

Reconstruction-Based

Closest generated image to $x$.

$$\mathcal{A}(x) = \mathbf{1}[\min_z \|x - G(z)\| < \tau]$$

Close reconstruction: likely memorized.

Hayes et al. 2017

Shadow GAN Approach

Stronger attacks: train shadow GANs.

  • Train $k$ shadow GANs on known data
  • Extract discriminator features (members / non-members)
  • Train binary attack classifier
  • Apply to target GAN's discriminator

Mode collapse increases vulnerability: poor coverage, stronger memorization.

Hayes et al. 2017

LOGAN: Results

DatasetGAN TypeAttack Accuracy
MNISTDCGAN~74%
CIFAR-10DCGAN~69%
CelebADCGAN~72%
  • Lower than classifier MIA (GANs leak less)
  • Still well above 50%
  • Smaller training sets → higher vulnerability
  • More mode collapse → higher vulnerability
  • Generative models also leak. MIA extends beyond classification.

Seq2Seq: Machine Translation

Sequence-model MIA (Hisamoto et al. 2020)

Hisamoto et al. 2019

MIA on Sequence-to-Sequence Models

Hisamoto et al.: MIA on machine translation.

Bridge from vision to NLP.

Challenge

No fixed-size confidence vector.

Variable-length output.

Solution

Perplexity on source-target pair as signal.

$$\text{PPL}(x, y) = \exp\left(-\frac{1}{T}\sum_{t=1}^T \log p_\theta(y_t \mid y_{<t}, x)\right)$$
Hisamoto et al. 2019

Perplexity as Membership Signal

Members

  • Seen pair
  • High probability
  • Low perplexity

Non-members

  • Unseen pair
  • Lower probability
  • Higher perplexity

Attack: $\mathcal{A}(x, y) = \mathbf{1}[\text{PPL}(x, y) < \tau]$

Same intuition as loss-threshold, in sequence-model form.

Hisamoto et al. 2019

Results & Implications for NLP

  • WMT translation models (English → German, etc.)
  • Perplexity attack: AUC well above 0.5
  • Smaller training sets → larger PPL gap
  • Fine-tuned models more vulnerable than from-scratch

First sequence-model MIA. Perplexity: baseline for all LLM MIA work.

Hisamoto et al. 2019

From Translation to Language Models

Seq2Seq MIALLM MIA (future)
Source-target pair $(x, y)$Text sequence $x = (x_1, \ldots, x_T)$
Conditional PPL: $p(y \mid x)$Unconditional PPL: $p(x)$
Small training sets (millions)Massive pre-training (trillions of tokens)

New challenges: unknown training distribution, calibration for text difficulty. (see notes)

Shadow Model Era: Key Takeaways

Shadow Models Work

Expensive.

Strong data and architecture assumptions.

Confidence Leaks

Rich signal.

More classes: more leakage.

Generative Models Too

GANs and seq2seq also vulnerable.

NLP Is Next

Seq2seq perplexity foreshadows LLM MIA.
