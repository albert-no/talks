Shokri et al. 2017

MIA against black-box ML-as-a-service

Shokri et al. 2017

Membership Inference Attacks Against ML Models

The first systematic black-box MIA against machine learning models.

  • Attacked commercial ML-as-a-service: Google Prediction API and Amazon ML
  • Black-box access: only observe prediction confidence vectors
  • Introduced the shadow model methodology
  • Up to 93% attack accuracy on some datasets

"Membership Inference Attacks Against Machine Learning Models"

IEEE S&P 2017

Shokri, Stronati, Song, Shmatikov

Shokri et al. 2017

Key Insight

Train shadow models that mimic the target model's behavior. Since you control the shadow models' training data, you can create a labeled dataset of (prediction, member/non-member) pairs to train an attack classifier.

The shadow model approach transforms a difficult unsupervised problem (does the model behave differently on members?) into a standard supervised classification problem.

Shokri et al. 2017

Shokri et al. 2017

Detailed Pipeline

  • Shadow training data: Generate datasets $D_1^s, \ldots, D_k^s$ from a distribution similar to the target's training data
  • Train shadow models: Train $k$ models $f_{\theta_1}, \ldots, f_{\theta_k}$ on these datasets
  • Collect predictions: For each shadow model, query it on its training data (label: "in") and on held-out data (label: "out")
  • Build attack dataset: $\{(\sigma(f_{\theta_j}(x)), \text{in/out})\}$ for all shadow models and queries
  • Train attack model: Binary classifier $\mathcal{A}_c$ (one per class $c$) on the attack dataset
  • Inference: Query target model on $x$, feed confidence to $\mathcal{A}_c$ for class $c = f_\theta(x)$
Shokri et al. 2017

Step 1: Shadow Model Training

Train $k$ shadow models that approximate the target's behavior:

Requirements

  • Data from a similar distribution as the target's training data
  • Same or similar model architecture
  • Same training procedure (epochs, optimizer, etc.)

Why It Works

  • Shadow models exhibit similar overfitting patterns
  • The gap between member and non-member confidence is a generalizable signal
  • Attack model learns "what membership looks like" across models
Shokri et al. 2017

Generating Shadow Training Data

Three strategies when the target's exact training distribution is unknown:

Model-Based

Query the target model to synthesize data that it classifies confidently. Use hill-climbing on the input space.

Statistics-Based

Use marginal statistics (feature distributions) from public data to sample synthetic records.

Noisy Real Data

Use data from a related domain or add noise to publicly available data from the same distribution.

Later work (Salem et al. 2019) will show that even unrelated data can work — the signal is more robust than initially thought.

Shokri et al. 2017

Step 2: Collecting Attack Training Data

For each shadow model $f_{\theta_j}$:

Member Queries

Query on training data $x \in D_j^s$

Record: $(\sigma(f_{\theta_j}(x)), \text{label}=\texttt{IN})$

Non-Member Queries

Query on held-out data $x \notin D_j^s$

Record: $(\sigma(f_{\theta_j}(x)), \text{label}=\texttt{OUT})$

The resulting attack dataset contains labeled examples of how shadow models behave on members vs. non-members.

Key assumption: the target model exhibits the same behavioral difference between members and non-members as the shadow models do.

Shokri et al. 2017

Step 3: Per-Class Attack Models

Train a separate binary classifier for each class $c$:

$$\mathcal{A}_c : \mathbb{R}^{|\mathcal{C}|} \to \{0, 1\}$$

Input: confidence vector $\sigma(f_\theta(x))$ for points predicted as class $c$

Output: member (1) or non-member (0)

Why per-class? Different classes have different confidence distributions. A model may be very confident on class "cat" but uncertain on class "bird" — the attack model needs class-specific calibration.

Shokri et al. 2017

Attack Inference

Given a target point $(x, y)$ and the target model $f_\theta$:

  • Query target: $\mathbf{p} = \sigma(f_\theta(x))$
  • Identify predicted class: $c = \arg\max_i \mathbf{p}_i$
  • Apply attack model: $\hat{m} = \mathcal{A}_c(\mathbf{p})$
  • If $\hat{m} = 1$: declare $x$ as a member

The attack classifier has learned to recognize the subtle differences in confidence patterns between members and non-members for each class.

Shokri et al. 2017

Results: Amazon ML

Similar results on Amazon's ML service, confirming the attack generalizes:

  • Effective across different ML platforms with different internal architectures
  • Works even when the attacker doesn't know the target's model type
  • Higher vulnerability with more output classes
  • Higher vulnerability with more overfitting (lower accuracy models)

Key finding: the attack is platform-agnostic. The behavioral signature of membership (high confidence on training data) is a universal property of overfitted models.

Shokri et al. 2017

Why More Classes = More Vulnerable

The output is a point on the probability simplex $\Delta^{C-1}$:

$$\sigma(f_\theta(x)) \in \Delta^{C-1} = \left\{\mathbf{p} \in \mathbb{R}^C : p_i \geq 0, \sum_i p_i = 1\right\}$$

Few classes (C=2)

Output is a single scalar (probability of class 1). Limited signal for the attacker.

Many classes (C=100)

Output is a 99-dimensional vector. The distribution of probability mass across classes reveals much more about how the model treats $x$.

More dimensions → richer behavioral fingerprint → easier to distinguish members from non-members.

Shokri et al. 2017

Limitations of Shadow Models

  • Data distribution assumption: Need access to data from a similar distribution as the target's training data
  • Architecture knowledge: Best results when shadow model architecture matches the target
  • Computational cost: Training $k$ shadow models is expensive ($k \times$ the cost of one model)
  • Per-class classifiers: Need separate attack models for each class — doesn't scale easily
  • No theoretical justification: Works empirically but lacks formal optimality guarantees

Open question: Can we do as well with fewer assumptions? (Spoiler: Yes — Salem et al., Sablayrolles et al.)

Shokri et al. 2017

Pseudocode: Shadow Model Attack

// Phase 1: Build attack training data for j = 1 to k: D_s[j] ← sample_data(distribution) f[j] ← train_model(D_s[j]) for x in D_s[j]: // members attack_data.add(f[j](x), "IN", class(x)) for x in holdout[j]: // non-members attack_data.add(f[j](x), "OUT", class(x)) // Phase 2: Train attack classifiers for c in classes: A[c] ← train_binary_classifier(attack_data[c]) // Phase 3: Attack def attack(target_model, x): p ← target_model(x) c ← argmax(p) return A[c](p) // 1 = member, 0 = non-member

The Simplest Attack: Threshold on Confidence

Before the full shadow model pipeline, consider the simplest possible attack:

$$\mathcal{A}(f_\theta, x) = \mathbf{1}\left[\max_c f_\theta(x)_c > \tau\right]$$

If the model is very confident on $x$, it was probably a training member.

Advantages

  • No shadow models needed
  • No training data needed
  • Single threshold to tune
  • Computationally free

Limitations

  • Global threshold ignores class differences
  • Doesn't use the full confidence vector
  • Sensitive to threshold choice
  • Weaker for well-calibrated models

Surprisingly, this simple baseline is often competitive with the full shadow model approach (Salem et al. 2019).

LOGAN: Generative Models

MIA for GANs (Hayes et al. 2019)

Hayes et al. 2017

LOGAN: MIA Against Generative Models

Shokri attacked classification models. Hayes et al. extended MIA to generative adversarial networks (GANs).

Challenge: GANs don't output prediction vectors. There's no natural "confidence score" to threshold on. How do we define membership for a generative model?

  • A GAN memorizes training images if it generates outputs very similar to specific training examples
  • The discriminator score can serve as a membership signal
  • Alternatively, reconstruction error can indicate membership
Hayes et al. 2017

LOGAN: Attack Architecture

Discriminator-Based

Use the GAN's discriminator $D(x)$ as a membership signal:

$$\mathcal{A}(x) = \mathbf{1}[D(x) > \tau]$$

The discriminator was trained to distinguish real (training) data from fake — it implicitly encodes membership.

Reconstruction-Based

Find the closest generated image to $x$:

$$\mathcal{A}(x) = \mathbf{1}[\min_z \|x - G(z)\| < \tau]$$

If the GAN can closely reconstruct $x$, it likely memorized it.

Hayes et al. 2017

Shadow GAN Approach

For more sophisticated attacks, train shadow GANs:

  • Train $k$ shadow GANs on known datasets
  • Extract discriminator features for members and non-members of each shadow
  • Train a binary attack classifier on these features
  • Apply to the target GAN's discriminator output

Key insight: mode collapse increases vulnerability. When a GAN fails to cover the full data distribution, it memorizes specific examples more strongly.

Hayes et al. 2017

Seq2Seq: Machine Translation

Toward sequence-model MIA (Hisamoto et al. 2020)

Hisamoto et al. 2019

MIA on Sequence-to-Sequence Models

Hisamoto et al. extended MIA to machine translation models — the bridge from vision to NLP.

Challenge

Seq2seq models don't output a fixed-size confidence vector. Instead, they generate variable-length sequences with per-token probabilities.

Solution

Use perplexity — the model's average surprise at the source-target pair — as the membership signal.

$$\text{PPL}(x, y) = \exp\left(-\frac{1}{T}\sum_{t=1}^T \log p_\theta(y_t \mid y_{<t}, x)\right)$$
Hisamoto et al. 2019

Perplexity as Membership Signal

Members (training pairs)

  • Model has seen this (source, target) pair
  • Assigns high probability to the correct translation
  • Low perplexity

Non-members

  • Model has not seen this pair
  • Assigns lower probability to the translation
  • Higher perplexity

Attack: $\mathcal{A}(x, y) = \mathbf{1}[\text{PPL}(x, y) < \tau]$

This is precisely the same intuition as the loss-based threshold attack for classifiers — just expressed in the language of sequence models.

Hisamoto et al. 2019

Results & Implications for NLP

  • Tested on WMT translation models (English → German, etc.)
  • Perplexity-based attack achieves significant AUC above 0.5
  • Smaller training sets → larger gap between member and non-member perplexity
  • Fine-tuned models more vulnerable than models trained from scratch

Significance: First demonstration that MIA extends naturally to sequence models — directly paving the way to LLM membership inference.

The perplexity-based approach will become the baseline for all LLM MIA work in Section VI.

Hisamoto et al. 2019

Shadow Model Era: Key Takeaways

Shadow Models Work

But they are expensive and require strong assumptions about data distribution and model architecture.

Confidence Vectors Leak

The prediction confidence is a rich signal for membership — more classes means more leakage.

Generative Models Too

GANs, seq2seq, and other generative architectures are also vulnerable to membership inference.

NLP Is Next

Perplexity-based attacks on seq2seq models directly foreshadow LLM MIA.

Open questions (circa 2017–2019): Can we attack without shadow models? What is the theoretically optimal attack? How does overfitting quantitatively relate to privacy?
