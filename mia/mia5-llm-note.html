Slides 180 + 180b — Practical LLM MIA (full pseudocode)

The deck splits these three attacks across two slides for readability. The full side-by-side pseudocode is reproduced here.

Perplexity baseline

def ppl_attack(model, text, τ):
    ppl = exp(-mean([log_prob(model, t, text[:i]) for i, t in text]))
    return ppl < τ

One scalar threshold on average per-token log-likelihood. No reference model, no shadow models. Cheap and easy to ship; saturates fast — high false-positive rate at low confidence because perplexity reflects text difficulty as much as memorization.

Neighbourhood attack

def neighbourhood_attack(model, ref_model, text, K):
    neighbours = [mask_and_fill(ref_model, text) for _ in range(K)]
    score = log_prob(model, text) - mean([log_prob(model, n) for n in neighbours])
    return score

Calibrate against locally-perturbed neighbours of the candidate text. Each neighbour is a small edit produced by a separate reference model (mask-and-fill). The attack score is the gap between target log-prob on the original text vs. on the neighbourhood — large positive gap implies memorization of the exact form. No shadow training required, which is what makes it usable on frozen pretrained LLMs.

Token-level InfoRMIA

def infomia_attack(target, ref, text):
    scores = []
    for t, token in enumerate(text):
        diff = log_prob(target, token, text[:t]) - log_prob(ref, token, text[:t])
        w = informativeness(ref, token, text[:t])
        scores.append(w * diff)
    return sum(scores)

Per-token weighted log-likelihood ratio between target and reference. The weight $w$ down-weights tokens the reference already explains (high reference probability, low informativeness) and up-weights tokens where the reference is uncertain — those are the tokens that carry the membership signal. Aggregating across the sequence with weights, rather than averaging perplexity, recovers most of the per-example calibration that LiRA achieves in the classification setting.

Reading the comparison

  • Perplexity conflates "memorized" with "easy to predict from the prefix" — it cannot distinguish memorization from intrinsic predictability.
  • Neighbourhood isolates "memorization of this exact wording" by comparing to neighbours that share semantic content but differ in surface form.
  • InfoRMIA reweights by token informativeness, surfacing memorization signal that perplexity averages away.
