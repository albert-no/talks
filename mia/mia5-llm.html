01

Why LLMs Are Different

Perplexity and its failures

Why LLMs Are Different

Massive Pre-training

Trillions of tokens from the web. Training data is unknown and never fully disclosed.

Autoregressive Generation

$p(x) = \prod_t p(x_t \mid x_{<t})$. No fixed output vector — variable-length sequences.

Layered Memorization

Pre-training + fine-tuning + RLHF creates multiple layers of memorization.

Scale

Billions of parameters. Training shadow LLMs is often infeasible.

Three LLM Threat Models

Pre-training Detection

"Was this text in the Pile / C4 / web crawl?"

Hardest: seen once among trillions of tokens. Weak signal.

Fine-tuning Detection

"Was this in the RLHF / instruction-tuning set?"

Moderate: multiple epochs on small data. Stronger signal.

Context Detection

"Was this in the prompt / retrieval context?"

Easiest: very recent exposure. Strong signal.

Most LLM MIA research focuses on pre-training (hardest) and fine-tuning detection.

The Perplexity Baseline

The simplest LLM MIA — threshold on perplexity (loss):

$$\text{PPL}(x) = \exp\left(-\frac{1}{T}\sum_{t=1}^T \log p_\theta(x_t \mid x_{<t})\right)$$

Attack: $\mathcal{A}(x) = \mathbf{1}[\text{PPL}(x) < \tau]$ — low perplexity implies membership.

Problem: "The cat sat on the mat" has low perplexity regardless of membership. Common text is always low-perplexity. Perplexity conflates memorization with general fluency.

Why Raw Perplexity Fails

Low PPL ≠ Member

  • "Hello, how are you?" — PPL ≈ 5, never in training
  • Common phrases, boilerplate text
  • Simple factual statements

High PPL ≠ Non-member

  • Technical jargon, rare names — PPL is high even for training text
  • Code, mathematical notation
  • Domain-specific terminology

The core challenge: Separate "this text is intrinsically easy/hard" from "this text was memorized." This is the calibration problem for LLM MIA.

Calibration: The Key Challenge

All modern LLM MIA methods are fundamentally calibration strategies:

MethodCalibration Approach
Reference model (zlib, smaller LM)Compare PPL against a different model
Neighbourhood (Mattern)Compare against synthetic paraphrases
Self-prompt (SPV-MIA)Compare against LLM's own generations
Context-aware (Chang)Measure variance across contexts
Token-level (InfoRMIA)Focus on informative tokens only

Each method answers: "Is this text's low perplexity due to memorization or intrinsic simplicity?"

02

Neighbourhood Attack

Mattern et al. 2023

Mattern et al. 2023

Neighbourhood MIA

Compare target text against synthetic neighbours (paraphrases):

If the text is memorized: PPL(original) $\ll$ PPL(neighbours).

If the text is just easy: PPL(original) $\approx$ PPL(neighbours).

The gap between original and neighbours isolates the memorization signal.

Mattern et al. 2023

Generating Neighbours

Use another language model to create paraphrases via mask-and-fill:

  • Take target text $x$
  • Randomly mask spans of tokens
  • Fill masked spans with another LM (e.g., T5)
  • Result: $\tilde{x}^{(k)} = \text{fill}(\text{mask}(x))$ for $k = 1, \ldots, K$

The neighbours preserve the meaning and structure but change specific words — creating counterfactual versions of the text.

Mattern et al. 2023

Neighbourhood Score

$$s(x) = \log p_\theta(x) - \frac{1}{K}\sum_{k=1}^K \log p_\theta(\tilde{x}^{(k)})$$

$s(x) \gg 0$ → Member

Original is suspiciously more likely than its paraphrases. The model has specific memory of this exact text.

$s(x) \approx 0$ → Non-member

Original is about as likely as paraphrases. The model treats all versions similarly — no specific memorization.

The difference cancels out intrinsic difficulty: easy text has low PPL for both original and neighbours.

Mattern et al. 2023

Neighbourhood: Results & Limitations

Results

  • Significant improvement over raw perplexity
  • Works on GPT-2, GPT-Neo
  • No reference models needed
  • Connection to counterfactual reasoning

Limitations

  • Quality of neighbours matters
  • Expensive: $K$ generations per target
  • Mask-and-fill may not preserve meaning for technical text
  • Requires access to a separate generation model

SPV-MIA and Context-Aware MIA

Reference-model strategies for LLMs

Fu et al. 2023

SPV-MIA: Self-Prompt Calibration

Use the target LLM itself to generate calibration data — no external models needed.

Key insight: If the model was fine-tuned on text $x$, then prompting the model to generate text "like $x$" will produce output very similar to $x$ — because $x$ is memorized.

For non-members, the self-generated text will be less similar to the target.

Fu et al. 2023

Self-Prompting Strategies

Prefix Completion

Give the first few tokens of $x$ as prompt. Compare the model's continuation with the actual text.

Paraphrase Generation

Ask the model to paraphrase $x$. Members produce paraphrases closer to the original.

Topic-Conditioned

Prompt with the topic of $x$. Members generate text in the same style/content as $x$.

$$\text{Score}(x) = \frac{p_\theta(x)}{p_\theta(\text{self-prompt}(x))}$$

The self-prompt acts as a personalized reference — no external reference model needed.

Fu et al. 2023

SPV-MIA: Results

MethodAUC (Fine-tuned LLaMA)AUC (Fine-tuned GPT-2)
Perplexity baseline~0.65~0.60
Neighbourhood~0.72~0.68
SPV-MIA~0.85~0.80
  • Superior for fine-tuning detection (strong memorization signal)
  • No external reference model needed
  • Less effective for pre-training detection (weaker memorization)
  • Requires generation capability (not just API logits)
Chang et al. 2024

Context-Aware MIA

Memorization in LLMs is context-dependent:

The same text is memorized differently depending on the surrounding context in which it appeared during training.

  • "To be or not to be" from Shakespeare's training data → memorized in Hamlet context
  • The same phrase in a random blog post → not memorized in that context
  • Previous MIA methods ignore context — they treat text as isolated
Chang et al. 2024

Context-Aware Attack Design

Provide different prefixes (contexts) and measure loss variation:

$$s(x) = \text{Var}_{c \in \mathcal{C}}\left[\log p_\theta(x \mid c)\right]$$

Members

With the right context, loss drops dramatically. With wrong context, it's higher. High variance.

Non-members

Loss is similar regardless of context — the model has no specific memory to activate. Low variance.

Context extraction: use beginning of document as prefix, topic-related prompts, or random contexts as baseline.

Chang et al. 2024

Context-Aware: Results & Implications

  • Significant improvement for pre-training data detection
  • Especially effective for longer documents with distinctive contexts
  • Complements other methods — can be combined with neighbourhood or token-level

Key implication: Memorization is not a property of text alone — it's a property of text-in-context. This has profound implications for data deletion and machine unlearning.

InfoRMIA

Information-theoretic token-level attack (2025)

Tao & Shokri 2025

Token-Level InfoRMIA

The key innovation: not all tokens are equally informative for membership inference.

  • Common words ("the", "is", "and") are predictable regardless of membership
  • Named entities, numbers, rare phrases are only predictable if memorized
  • Focus on informative tokens → dramatically stronger signal

"Stronger Membership Inference and Memorization Assessment for LLMs"

Tao & Shokri, 2025

Tao & Shokri 2025

Token-Level Surprise

Define the surprise (information content) at each token position:

$$I_t = -\log p_\theta(x_t \mid x_{<t})$$

Low Surprise Tokens

"The", "is", "a" — predictable from context alone. Uninformative for MIA.

High Surprise Tokens

Named entities, specific numbers, rare words — only predictable if memorized. Highly informative.

Tao & Shokri 2025

InfoRMIA: Formulation

$$s(x) = \sum_{t \in \mathcal{T}_{\text{info}}} w_t \cdot \left(\log p_\theta(x_t \mid x_{<t}) - \log p_{\text{ref}}(x_t \mid x_{<t})\right)$$
  • $\mathcal{T}_{\text{info}}$ = set of informative tokens (high surprise under reference)
  • $w_t$ = weight reflecting token informativeness
  • $p_\theta$ = target model, $p_{\text{ref}}$ = reference model
  • The difference per token cancels intrinsic difficulty (like neighbourhood, but at token level)
Tao & Shokri 2025

Identifying Informative Tokens

Strategies to find $\mathcal{T}_{\text{info}}$:

High Reference Surprise

Tokens where reference model is surprised → target's low surprise indicates memorization.

Named Entities

People, places, organizations — specific facts that require memorization.

Target-Reference Gap

Tokens where target and reference disagree most — maximum membership signal.

Information-Theoretic Foundation

Connection to mutual information:

$$I(z \in D \;;\; x_t \mid x_{<t})$$

The mutual information between membership and the token prediction at position $t$.

  • Tokens with high MI contribute most to the attack
  • The weighted sum over informative tokens maximizes the total MI
  • Provides a formal bound on attack power from token-level MI

InfoRMIA is the token-level analogue of the likelihood ratio: it identifies where in the text the membership signal lives.

Tao & Shokri 2025

MethodAUC (WikiMIA)TPR@1% FPR

InfoRMIA achieves state-of-the-art on pre-training detection — the hardest LLM MIA setting.

Token-level analysis also enables targeted unlearning: identify exactly which tokens to "forget."

Benchmarks: WikiMIA & MIMIR

WikiMIA

Temporal split on Wikipedia: articles before vs. after the model's training cutoff date.

Clean ground truth but limited to Wikipedia style.

MIMIR

Multi-domain evaluation: Wikipedia, books, news, code, academic papers.

More diverse but harder ground truth.

Standardized benchmarks enable fair comparison, but ground truth is always approximate — we never know the full training data of proprietary LLMs.

Scale Effects on MIA Vulnerability

How does model size affect MIA vulnerability?

  • Larger models generalize better → smaller generalization gap → less MIA by Yeom's bound
  • But larger models also memorize more → more data is extractable (Carlini et al.)
  • Net effect: larger models may have lower average vulnerability but higher worst-case vulnerability

Scaling law of memorization: The amount of memorized content grows with model size and training repetitions. More parameters = more capacity to store specific training examples.

From MIA to Extraction

Stronger attacks built on the same foundation

From MIA to Training Data Extraction

MIA is a precursor to the stronger attack of data extraction:

MIA:
Is $x$ a member?
→
Extraction:
Generate $x$ from model
→
Verify:
Is generated $x$
from training?
  • Carlini et al. (2021): Extracted 600+ verbatim training sequences from GPT-2
  • MIA serves as the verification step: was the generated text actually in training?
  • The full attack chain: generate candidates → rank by MIA score → extract highest-scoring

Extractable vs. Inferable Memorization

Extractable

The model can be prompted to generate verbatim training text. Directly produces the memorized content.

Stronger attack, harder to achieve.

Inferable

Given a text, the model's behavior reveals whether it was trained on it. Doesn't produce the text itself.

Weaker attack (MIA), but more broadly applicable.

Inferable memorization is a superset of extractable — everything extractable is inferable, but not vice versa.

Practical LLM MIA: Pseudocode

// Perplexity Baseline def ppl_attack(model, text, τ): ppl ← exp(-mean([log_prob(model, t, text[:i]) for i,t in text])) return ppl < τ // Neighbourhood Attack def neighbourhood_attack(model, ref_model, text, K): neighbours ← [mask_and_fill(ref_model, text) for _ in range(K)] score ← log_prob(model, text) - mean([log_prob(model, n) for n in neighbours]) return score // Token-Level InfoRMIA def infomia_attack(target, ref, text): scores ← [] for t, token in enumerate(text): diff ← log_prob(target, token, text[:t]) - log_prob(ref, token, text[:t]) w ← informativeness(ref, token, text[:t]) scores.append(w * diff) return sum(scores)

LLM Defenses and Governance

DP fine-tuning, deduplication, unlearning, law

LLM Defenses: DP Fine-Tuning

Apply DP-SGD during fine-tuning to limit memorization:

  • DP-SGD for fine-tuning is more practical than for pre-training (smaller dataset, fewer steps)
  • Per-example gradient clipping + noise injection
  • Utility cost is significant but manageable for fine-tuning
  • Libraries: Opacus (PyTorch), TensorFlow Privacy

DP for pre-training at LLM scale remains an open challenge — the utility-privacy tradeoff is too severe for trillion-token training.

Deduplication & Data Curation

Reducing memorization through training data quality:

Deduplication

Remove repeated text from the training corpus. Repeated text is memorized much more strongly.

Carlini et al. showed: memorization scales with repetition count.

Data Curation

Filter out PII, unique identifiers, and sensitive content before training.

Prevents the most harmful memorization but doesn't eliminate MIA vulnerability.

Machine Unlearning for LLMs

Remove a data point's influence from a trained model without full retraining:

Trained Model
→
Unlearn $x$
→
Updated Model
→
Verify with MIA
  • MIA is the verification tool: did unlearning actually work?
  • Current unlearning methods often fail MIA tests — the "forgotten" data is still detectable
  • InfoRMIA's token-level analysis enables targeted unlearning — know exactly which tokens to forget

Legal & Ethical Dimensions

GDPR Right to Erasure

Article 17: individuals can request deletion of their data. But how do you "delete" from a trained model?

MIA provides the test: if MIA still detects the data, it hasn't been effectively deleted.

NYT v. OpenAI

Demonstrated verbatim reproduction of NYT articles from ChatGPT.

MIA as legal evidence: proving specific content was in training data.

MIA as a Tool for Data Governance

  • Opt-out verification: Did the model respect my data removal request?
  • Copyright compliance: Was copyrighted content used for training?
  • Regulatory audit: Does this model meet privacy requirements?
  • Responsible AI: How much private data did the model memorize?

MIA bridges the gap between legal requirements (GDPR, copyright) and technical verification. It's the closest we have to a practical test for "did you use my data?"

Key Themes Across the Decade

1. Overfitting Is the Root Cause

From Yeom's bound to LLM memorization: the gap between training and population behavior is what MIA exploits.

2. Black-Box ≈ White-Box

The loss contains nearly all membership information. Gradients and activations add negligible signal.

3. Per-Example Calibration

Global thresholds are suboptimal. The best attacks calibrate for each data point's difficulty.

4. Finer Granularity

From sequence-level → document-level → token-level. The trend is toward more precise localization of memorization.

Open Problem: Pre-Training Detection

Detecting membership in pre-training data remains the hardest open problem:

  • Weak signal: text seen once among trillions of tokens
  • Current best: ~70–75% AUC (far from perfect)
  • Unknown training data makes ground truth approximate
  • Scaling to larger models hasn't clearly improved detection

Needs: Better calibration, multi-signal fusion, and improved benchmarks with verified ground truth.

Open Problem: Multi-Modal MIA

MIA for vision-language models (CLIP, GPT-4V, Gemini):

  • Image-text pairs create richer membership signal than either modality alone
  • Cross-modal leakage: can you infer text membership from image behavior?
  • Multimodal models are trained on web-scale image-text pairs — massive attack surface
  • Almost entirely unexplored in the literature

Open Problem: Reference-Free MIA

Can we achieve LiRA-level performance with zero reference models?

  • SPV-MIA uses self-generation — a step toward reference-free
  • Neighbourhood uses an auxiliary model for paraphrasing
  • True zero-resource MIA (only the target model) remains limited to threshold attacks
  • Information-theoretic bounds on reference-free attack power are unknown

Open Problem: MIA for Agents & RAG

New threat models for compound AI systems:

RAG Systems

Is a retrieved document from the training set or the retrieval corpus? MIA must distinguish pre-trained knowledge from retrieved context.

LLM Agents

Agents with memory and tool use. Membership in what? Training data, conversation history, or tool outputs?

These systems blur the line between "training data" and "operational data" — new threat models are needed.

Practical Recommendations

For ML Practitioners

  • Evaluate MIA vulnerability before deployment
  • Use Privacy Meter or similar tools
  • Apply DP if privacy is critical
  • Deduplicate training data
  • Monitor for extraction attempts

For Researchers

  • Report TPR at low FPR, not just AUC
  • Test against adaptive attacks
  • Evaluate per-sample risk
  • Use standardized benchmarks
  • Report computational cost
