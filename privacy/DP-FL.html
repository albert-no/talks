Train a shared model across many clients without moving raw data off-device.

  1. Each client computes a local gradient.
  2. Clients upload gradients to the server.
  3. Server aggregates and updates the model.
  4. Server broadcasts the update back.

Three Challenges in FL

Compression

Millions of clients, millions of parameters. Uplink bandwidth is the bottleneck.

Privacy

Gradients leak training data. Need formal guarantees on each user's contribution.

Utility

After compression and privatization, the aggregate must still train a useful model.

This talk: the three-way tradeoff between these axes.

The Core Primitive

Each of $n$ clients holds $x_{i} \in \mathbb{R}^{d}$ with $\|x_{i}\| \le 1$. The server estimates:

$$\hat\mu \;\approx\; \frac{1}{n}\sum_{i} x_i$$
  • Compression: each message uses at most $b$ bits.
  • Privacy: each message is $\varepsilon$-LDP.
  • Utility: minimize $\mathbb{E}\|\hat\mu - \mu\|^2$.
A gradient step is a mean of per-client gradients. Nail mean estimation, nail DP-SGD in FL.
02

LDP vs. Central DP

Different trust models, different achievable rates.

Differential Privacy

Definition (Dwork, McSherry, Nissim, Smith, 2006).

A randomized mechanism $M$ is $(\varepsilon, \delta)$-DP if for any two datasets $D, D'$ differing in one record, and any event $S$:

$$\Pr[M(D)\in S] \le e^{\varepsilon}\cdot \Pr[M(D')\in S] + \delta$$
  • Small $\varepsilon$: output distribution barely shifts when one person's data changes.
  • Post-processing immunity, strong composition, group-privacy scaling.
  • Where it runs matters: raw data vs. each user's own device.
Dwork, McSherry, Nissim, Smith, "Calibrating Noise to Sensitivity in Private Data Analysis", TCC 2006.

Local DP, Formally

Definition (Local Differential Privacy).

Client $i$ runs its own randomized mechanism $Q(\cdot \mid x_i)$ satisfying:

$$\frac{Q(y\mid x)}{Q(y\mid x')} \le e^{\varepsilon} \quad \text{for all } x, x', y.$$

Why it is hard

No trusted aggregator. Every outgoing message must already be $\varepsilon$-LDP, so noise accumulates across users instead of canceling.

Why it is required

When the aggregator is untrusted or could be compelled to disclose, central DP is not enough. LDP gives each user plausible deniability before their message leaves the device.

LDP Mean Estimation: Minimax Rate

$n$ clients each hold $x_i \in \mathbb{R}^d$ with $\|x_i\| \le 1$; each sends an $\varepsilon$-LDP message $Y_i = M(x_i)$.

Theorem (Duchi, Jordan, Wainwright, 2013).

The minimax MSE for $\varepsilon$-LDP mean estimation is:

$$\Theta\!\left(\frac{d}{n \cdot \min(\varepsilon,\varepsilon^2)}\right).$$
Compare central DP: $\Theta(1/(n^2\varepsilon^2))$. LDP pays an extra factor of $d$ — the price of no trusted aggregator.
Duchi, Jordan, Wainwright, "Local Privacy and Statistical Minimax Rates", FOCS 2013.

Gaussian Mechanism is Suboptimal for LDP

Natural first try: $M(x) = x + \mathcal{N}(0,\sigma^2 I_d)$. Works well for central DP — but under LDP wastes privacy budget along irrelevant directions. MSE scales as $d/\varepsilon^2$, off by a factor linear in $d$ in the high-privacy regime.

PrivUnit: Optimal LDP Mechanism

PrivUnit (Bhowmick et al., 2018).

For input $v \in \mathbb{S}^{d-1}$, sample $W \sim \mathrm{Uni}(\mathbb{S}^{d-1})$. Choose $(p, \gamma)$ so the density ratio equals $e^{\varepsilon}$:

$$\mathrm{PrivUnit}(p,\gamma) \sim \begin{cases} W \mid \langle W, v\rangle \ge \gamma & \text{w.p. } p \\ W \mid \langle W, v\rangle < \gamma & \text{w.p. } 1{-}p \end{cases}$$
  • Output stays on $\mathbb{S}^{d-1}$: no wasted Gaussian tails.
  • Achieves exact optimal constant for $\ell_2$ LDP mean estimation.
Asi, Feldman, Talwar, "Optimal Algorithms for Mean Estimation under Local Differential Privacy", ICML 2022.

The Missing Axis: Communication

PrivUnit is optimal in privacy, but its output is a continuous point on the sphere — requires many bits to transmit.

What was known

  • Privacy alone: PrivUnit optimal.
  • Communication alone: randomized coding optimal.
  • Both together? Open until our work.

Prior work (rate-matching only)

  • SQKR — Chen, Kairouz, Ozgur, NeurIPS 2020
  • FT21 — Feldman, Talwar, ICML 2021
  • MMRC — Shah et al., AISTATS 2022

All match the right rate but none are exactly optimal.

03

Exact Optimality

NeurIPS 2023: privacy, communication, and utility — exactly.

Problem Setup

Goal: jointly optimize rate, utility, and privacy.

  • Rate: minimum bits per user.
  • Utility: minimum MSE on the aggregate.
  • Privacy: each message is $\varepsilon$-LDP.

Clients share common randomness (public seed) with the server. Free in practice — and, as we show, very powerful.

Protocol parameters

  • $x_{i} \in \mathbb{S}^{d-1}$
  • Budget: $\varepsilon$ LDP, $b$ bits
  • Shared seed $U$

Output: $\hat\mu$ minimizing $\mathbb{E}\|\hat\mu - \mu\|^2$

Isik, Chen, Ozgur, Weissman, No, "Exact Optimality of Communication-Privacy-Utility Tradeoffs in Distributed Mean Estimation", NeurIPS 2023.

LDP with Shared Randomness

Shared randomness does not weaken privacy. The seed $U$ is public, so privacy is measured conditional on $U$.

  • Client encoder: $m = f(x, U)$, at most $b$ bits.
  • Server decoder: $\hat{v} = g(m, U)$.
  • For every fixed $U$: the map $x \mapsto m$ must be $\varepsilon$-LDP.
A shared codebook lets every bit carry information about direction, not spend on noise.

Result I: Canonical Protocols

Theorem (Canonical reduction).

Without loss of optimality, restrict to protocols of the form:

  • Single encoder: all $n$ clients use the same randomized encoder.
  • Single decoder: server decodes each $Y_i$ separately, then averages.
  • Unbiased: $\mathbb{E}[\mathrm{Dec}(Y) \mid x] = x$.
Designing a good single-user scheme suffices. The $n$-user MSE then scales as $(1/n) \cdot \mathrm{Var}_{\text{single}}$.

Result II: Codebook Schemes are Optimal

For any canonical, unbiased, $\varepsilon$-LDP protocol $(f, g, P_U)$, there exists a codebook-based scheme achieving the same error.

$$\text{Codebook}\quad U^{M} = (U_1, \ldots, U_M) \in (\mathbb{R}^d)^M$$ $$\text{Encoder: } m = f(v, U^{M}), \qquad \text{Decoder: } \hat v = U_m$$
  • Codebook drawn from the shared seed — known to client and server.
  • Client sends one index $m \in [M]$: costs $\lceil \log_{2} M \rceil$ bits.
From coding theory: the index $m$ identifies a codeword; privacy is enforced at the encoder, not at the output.

Result III: Rotationally Symmetric Simplex Codebook

Take a regular $(M{-}1)$-simplex $\{s_1, \ldots, s_M\}$ inscribed in $\mathbb{S}^{d-1}$, then Haar-rotate it:

$$(s_i)_j = \begin{cases} \dfrac{M-1}{\sqrt{M(M-1)}} & i = j \\ -\dfrac{1}{\sqrt{M(M-1)}} & i \ne j,\ j \le M \\ 0 & j > M \end{cases}$$
$$(U_1, \ldots, U_M) \stackrel{d}{=} (rA s_1, \ldots, rA s_M)$$

where $A \sim \mathrm{Haar}(\mathrm{O}(d))$ and $r$ is chosen so the decoder is unbiased.

Result IV: k-Closest Encoding is Optimal

Under RRSC, the optimal encoder $Q_f(m \mid v, U^{M})$ is two-level:

Theorem (k-closest optimality).

$$Q_f(m \mid v, U^{M}) = \begin{cases} \dfrac{e^{\varepsilon}}{k e^{\varepsilon} + M - k} & m \in \text{top-}k \\ \dfrac{1}{k e^{\varepsilon} + M - k} & \text{otherwise} \end{cases}$$
  • Density ratio $= e^{\varepsilon}$: the encoder is exactly $\varepsilon$-LDP.
  • Just a top-$k$ query on the codebook; optimal $k = k(\varepsilon, M)$.

RRSC Converges to PrivUnit

As $M \to \infty$, RRSC + $k$-closest $\longrightarrow$ PrivUnit.

  • As $M$ grows, rotated simplex vertices densify uniformly on $\mathbb{S}^{d-1}$.
  • Top-$k$ codewords converge to a spherical cap around $v$.
  • Two-level mass keeps density ratio at exactly $e^{\varepsilon}$: that is PrivUnit.

A Unified Framework

SQKR — NeurIPS 2020

Chen, Kairouz, Ozgur. First matching-rate bounds using $\Theta(\varepsilon)$ bits with shared randomness.

FT21 — ICML 2021

Feldman, Talwar. Simulating an LDP mechanism in $\Theta(\varepsilon)$ bits via channel simulation.

MMRC — AISTATS 2022

Shah et al. Minimal random coding — simulates PrivUnit with shared randomness.

RRSC — NeurIPS 2023 (this talk)

Isik, Chen, Ozgur, Weissman, No. RRSC + $k$-closest matches the PrivUnit lower bound exactly, with finite bits.

Experiments

RRSC vs. SQKR, MMRC, PrivUnitG on synthetic mean estimation over $\mathbb{S}^{d-1}$, varying $\varepsilon$, $b$, and $n$. RRSC matches PrivUnit utility at a fraction of the bits.

Open Question

Is RRSC + $k$-closest exactly optimal among all canonical protocols, not just rotationally-symmetric codebooks?

Conjecture. Yes: any exactly-optimal finite-bit LDP mean-estimation protocol is equivalent, up to post-processing, to RRSC with some $M$.
  • Extensions to heavy-tailed $x_i$ and to shuffled DP are open.
  • Adaptive $M$ per round?
04

DP in Modern ML

From mean estimation to realistic generative models.

From Primitives to Real Models

Clipping, private aggregation, and privacy accounting plug directly into training pipelines for real models.

  • DP-SGD: clip per-example gradients, add Gaussian noise, account the privacy cost.
  • Key trick: large public pre-training, then private fine-tuning on sensitive data.
  • Bigger backbones absorb noise better; better accountants (PRV, f-DP) tighten bounds.
Public pre-training gives a strong prior — spending the privacy budget on fine-tuning only.

DP-Diffusion

Ghalebikesabi et al. (2023). Diffusion models trained with DP-SGD can still produce high-quality samples.

  • Pretrain on public data; fine-tune on sensitive data with DP-SGD.
  • Per-example gradient clipping over the noise-prediction loss at each diffusion step.
  • State-of-the-art FID for DP image generation on CIFAR-10 and Camelyon17.
2023 — Google DeepMind

Differentially Private Diffusion Models Generate Useful Synthetic Images

First DP synthetic data strong enough to train downstream classifiers near par with real data.

Ghalebikesabi et al., "Differentially Private Diffusion Models Generate Useful Synthetic Images", arXiv:2210.09929, 2023.

DP-RDM: Retrieval-Augmented DP Generation

Lebensold et al. (2024). Instead of fine-tuning the backbone with DP-SGD, make the retrieval step differentially private.

  • Keep a public diffusion model frozen.
  • Retrieve from a private datastore using DP aggregation of retrieved embeddings.
  • Strong sample quality without fully privatizing backbone training.

The embedding average at retrieval time is exactly a mean-estimation problem — directly applicable to our framework.

The generator is public; privacy is spent only on the retrieval aggregate.

Sampled Gaussian Mechanism

Private data enters only via the noisy aggregate $z$, a mean of $k$ retrieved embeddings on a random subsample:

$$z = \tfrac{1}{k}\sum_{j=1}^{k} N_k^{(j)} + \mathcal{N}(0, \sigma^2 I)$$

DP-RDM Intuition

Private data influences the output only through retrieval. Make that one channel DP — post-processing through a frozen $\phi$ stays DP — no DP fine-tuning of the backbone.

Why DP-SGD is avoided

DP-SGD needs huge batches and retraining from scratch. DP-RDM freezes $\phi$ and spends privacy budget only on the retrieval mean.

The $\lambda$ knob

$\lambda = 0$: rely only on public neighbors $O_k$ — no privacy cost, domain gap. $\lambda \to 1$: rely on noisy private $z$ — more adaptation, more noise. The SGM accountant composes the $(\alpha, \varepsilon)$ cost over queries.
