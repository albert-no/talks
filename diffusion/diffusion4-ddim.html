01

What the DDPM Loss Sees

Freedom to redesign the forward process

DDPM $\epsilon$-Loss Recap

Per-step training loss:

$$L_{n-1} = \tilde{\lambda}_n\,\mathbb{E}_{X^{(0)},\,\epsilon}\!\left[\bigl\|\epsilon - \epsilon_\theta\bigl(\sqrt{\bar{\alpha}_n}\,X^{(0)} + \sqrt{1-\bar{\alpha}_n}\,\epsilon,\; n\bigr)\bigr\|^2\right].$$

Input to $\epsilon_\theta$: a sample from the single-time marginal.

Only Marginals Enter

The forward process appears only through:

$$q\bigl(X^{(n)}\,|\,X^{(0)}\bigr) = \mathcal{N}\!\bigl(\sqrt{\bar{\alpha}_n}\,X^{(0)},\;(1-\bar{\alpha}_n)\,I\bigr).$$

The joint $q(X^{(1:N)}\,|\,X^{(0)})$ never appears in training.

Any joint with the same marginals yields the same loss.

DDIM in One Sentence

Keep the DDPM marginals. Build a non-Markovian joint with parameter $\{\sigma_n\}$.

Recover a sampler with tunable stochasticity, down to fully deterministic.

Same trained network. Different sampler.

Song, Meng & Ermon, "Denoising Diffusion Implicit Models", ICLR 2021.
02

Non-Markovian Forward

Same marginals, tunable stochasticity

Setup

  • Schedule $\{\bar{\alpha}_n\}_{n=0}^N$ inherited from DDPM.
  • Free parameters $\{\sigma_n \geq 0\}_{n=1}^N$.
  • Target: define $q$ on $X^{(0:N)}$ with the DDPM marginals.

Strategy: specify $q(X^{(n)}\,|\,X^{(n+1)}, X^{(0)})$, condition both on the next step and the clean signal.

Forward Conditional

For $n = 0, \ldots, N-1$:

$$q(X^{(n)}\,|\,X^{(n+1)}, X^{(0)}) = \mathcal{N}\!\left(\sqrt{\bar{\alpha}_n}\,X^{(0)} + \frac{\sqrt{1-\bar{\alpha}_n - \sigma_{n+1}^2}}{\sqrt{1-\bar{\alpha}_{n+1}}}\,\bigl(X^{(n+1)} - \sqrt{\bar{\alpha}_{n+1}}\,X^{(0)}\bigr),\;\sigma_{n+1}^2\,I\right).$$

The mean depends on both $X^{(n+1)}$ and $X^{(0)}$.

Conditioning on $X^{(0)}$ everywhere makes the chain non-Markovian in $X^{(n)}$ alone.

Marginals Match DDPM

For every $n$ and every choice of $\{\sigma_n\}$:

$$q\bigl(X^{(n)}\,|\,X^{(0)}\bigr) = \mathcal{N}\!\bigl(\sqrt{\bar{\alpha}_n}\,X^{(0)},\;(1-\bar{\alpha}_n)\,I\bigr).$$

Identical to the DDPM single-time marginal.

A DDPM-trained $\epsilon_\theta$ is directly usable in DDIM.

Proof by Backward Induction

Base case $n=N$: by construction.

Inductive step: assume $X^{(n+1)}\,|\,X^{(0)} \sim \mathcal{N}\!\bigl(\sqrt{\bar{\alpha}_{n+1}}\,X^{(0)},\;(1-\bar{\alpha}_{n+1})\,I\bigr)$. Sum of two independent Gaussians conditional on $X^{(0)}$:

$$\begin{aligned} \mathbb{E}[X^{(n)}|X^{(0)}] &= \sqrt{\bar{\alpha}_n}\,X^{(0)}, \\ \mathrm{Var}(X^{(n)}|X^{(0)}) &= (1-\bar{\alpha}_n - \sigma_{n+1}^2) + \sigma_{n+1}^2 = 1-\bar{\alpha}_n. \end{aligned}$$

$\sigma_{n+1}^2$ cancels: marginals independent of $\{\sigma_n\}$.

DDPM as a Special Case

The DDIM family contains DDPM. Pick:

$$\sigma_{n+1}^2 = \beta_{n+1}\,\frac{1-\bar{\alpha}_n}{1-\bar{\alpha}_{n+1}}.$$

Forward collapses to $q(X^{(n)}\,|\,X^{(n+1)})$: Markov. Recovers the DDPM reverse step.

Exercise. Verify this $\sigma$ choice gives the DDPM update.

$\sigma_n \to 0$: Deterministic Forward

Set every $\sigma_n = 0$. Forward conditional becomes Dirac.

Given $X^{(0)}$ and one $X^{(n)}$, all other $X^{(k)}$ are determined. Marginals still match DDPM.

One knob $\{\sigma_n\}$ interpolates DDPM (full noise) to fully deterministic.

Exercise. Show the $\sigma_n=0$ update is a consistent discretization of the probability-flow ODE.

03

DDIM Sampling

Predicted clean signal, ODE limit

Training: Unchanged

DDIM trains the same $\epsilon$-prediction network as DDPM.

Sample $X^{(0)} \sim p_{\text{data}}$, $\epsilon \sim \mathcal{N}(0,I)$, $n \sim \mathrm{Unif}$:

$$X^{(n)} = \sqrt{\bar{\alpha}_n}\,X^{(0)} + \sqrt{1-\bar{\alpha}_n}\,\epsilon.$$

Loss depends only on single-time marginals.

The non-Markovian structure pays off only at sampling time.

Predicted Clean Signal

Invert the noise equation $X^{(n+1)} = \sqrt{\bar{\alpha}_{n+1}}\,X^{(0)} + \sqrt{1-\bar{\alpha}_{n+1}}\,\epsilon$ for $X^{(0)}$, plug in $\epsilon_\theta$:

$$\hat{X}^{(0)} = \frac{X^{(n+1)} - \sqrt{1-\bar{\alpha}_{n+1}}\,\epsilon_\theta(X^{(n+1)}, n+1)}{\sqrt{\bar{\alpha}_{n+1}}}.$$

The network's running guess for the clean sample.

DDIM Reverse Update

Forward conditional with $X^{(0)} \to \hat{X}^{(0)}$ and $X^{(n+1)}-\sqrt{\bar{\alpha}_{n+1}}\hat{X}^{(0)} \to \sqrt{1-\bar{\alpha}_{n+1}}\,\epsilon_\theta$:

$$X^{(n)} = \underbrace{\sqrt{\bar{\alpha}_n}\,\hat{X}^{(0)}}_{\text{toward data}} + \underbrace{\sqrt{1-\bar{\alpha}_n-\sigma_{n+1}^2}\,\epsilon_\theta(X^{(n+1)}, n+1)}_{\text{noise direction}} + \underbrace{\sigma_n\,Z^{(n)}}_{\text{fresh noise}}.$$

Three pieces: signal pull, noise direction, optional fresh noise.

Sampling Algorithm

DDIM Sampling
  1. Sample $X^{(N)} \sim \mathcal{N}(0, I)$.
  2. For $n = N{-}1, \ldots, 0$:
    1. $\hat{X}^{(0)} = \bigl(X^{(n+1)} - \sqrt{1-\bar{\alpha}_{n+1}}\,\epsilon_\theta(X^{(n+1)}, n+1)\bigr)/\sqrt{\bar{\alpha}_{n+1}}$.
    2. Sample $Z^{(n)} \sim \mathcal{N}(0,I)$ if $\sigma_n > 0$.
    3. $X^{(n)} = \sqrt{\bar{\alpha}_n}\,\hat{X}^{(0)} + \sqrt{1-\bar{\alpha}_n-\sigma_{n+1}^2}\,\epsilon_\theta(X^{(n+1)}, n+1) + \sigma_n\,Z^{(n)}$.
  3. Return $X^{(0)}$.

$\sigma_n = 0$: skip step (b), generator is deterministic.

Probability-Flow ODE

At $\sigma_n \equiv 0$, the reverse step is deterministic. Continuous-time limit:

$$\frac{dX_t}{dt} = f(X_t, t) - \tfrac{1}{2}g^2(t)\,\nabla_{\mathbf{x}} \log p_t(X_t).$$

Same marginals $\{p_t\}$ as the forward SDE. DDIM at $\sigma_n=0$ is a discretization for VP diffusion.

Stochastic forward, deterministic generator. Score does all the work.

04

Consequences of Determinism

Why DDIM matters in practice

Three Benefits

Fewer steps

Large strides on the index axis.

50 steps match 1000-step DDPM.

Inversion

Run the generator backwards.

Recover latent noise from an image.

Latent interpolation

Smooth paths in image space.

From interpolated noise latents.

All three rely on $X^{(0)}$ being a deterministic function of $X^{(N)}$.

Lecture Recap

  • DDPM $\epsilon$-loss depends only on $q(X^{(n)}\,|\,X^{(0)})$.
  • DDIM: non-Markovian forward, same marginals, parameter $\{\sigma_n\}$.
  • $\sigma_n \equiv 0$: deterministic generator, probability-flow ODE.
  • Training shared with DDPM. Sampling is a solver choice.
  • Wins: fewer steps, invertibility, clean latent interpolation.
